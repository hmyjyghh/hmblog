    # 类与对象

    • OOP(Object-Oriented Programming): 面向对象编程
    • 面向对象的主要思想: 也是一种代码封装的方法

    # 使用class 关键字定义一个类

    创建步骤

    1. 首先需要创建一个类
    2. 再通过类，创建实际的对象(可以类比构造函数，也可以类比模具，先有模具，再进行批量生产。)
    • 类名，首字母要大写
    • 对象 = 属性 + 方法

    # 多重继承

    也就是一个子类，可以同时继承多个父类

    class A:
    name = 'hmyjy'
    
    def hello(self):
        print('hello, hmyjy~')


class B:
    name = "happy year"
    y = 520

    def hello(self):
        print("hello, new year")


class C(A, B):
    pass

c1 = C()

print(c1.name) # hmyjy
print(c1.hello())
# hello, hmyjy~
# None

    通过上面的示例，可以发现

    • 继承多个类，可见访问顺序是从左到右，如果在A这个父类里，找到了，则不会再去B类里找
    • 什么时候会去B类里找呢？eg: print(c1.y), 只有在当前类(C这个类)找不到了，然后左侧的父类A也没找到, 才会去父类B中寻找, 从B类里，找到y的值
    • 因为A类里面没有y, 所以会去B类里找

    # 组合

    • 花园和动物之间，并不是从属关系, 没有谁属于谁的说法
    • 这种时候就应该使用组合，而不是继承
    • 做法，就是直接将相关的实例放到花园的类里边就可以了

    # 类相关知识点

    1. class B(A):, 括号里面指定它要继承的父类A
    2. class B(A):, 代表这个类B就是继承自类A的
    3. 如果类B里面，存在跟类A一样的属性和方法，会是：子类覆盖B类，同局部变量和全局变量的这种覆盖
    4. 判断一个对象是否属于某个类: isinstance()函数
    5. isinstance()函数, 是一个BIF
    print(isinstance(b, B)) # True
print(isinstance(b, A)) # True
    1. issubclass()函数, 是用于检测一个类是否为某个类的子类
    print(issubclass(A, B)) # False
print(issubclass(B, A)) # True

    # 封装

    1. 封装就是：在创建对象之前，将相关的属性和方法打包到一起
    2. 然后通过类来生成响应的对象
    3. 面向对象编程的三个基本特征： 封装、继承、多态

    # 继承

    1. 它可以使用现有类的所有功能, 并在无需重新编写代码的情况下,对这些功能进行扩展
    2. 通过继承创建的新类，我们称之为子类, 而被继承的类，我们称之为父类(基类或者超类)

    • 因为同一个类它是可以生成无数个对象的

    • Python 如何知道到底是哪个対象在调用呢, 就是通过这个self 参数传递的信息, 所以类中的每一个方法，默认的第一个参数都是 self

    # 重写

    • 如果对于父类的某个属性，或某个方法不满意的话，就可以重写一个同名的属性 或者 方法 对其进行覆盖。
    • 这种，我们就称之为：子类对父类的重写。
    class C:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def add(self):
        return self.x + self.y
    
    def mul(self):
        return self.x * self.y


class D(C):

    # 重写示例二

    • C.__init__(x, y), 这种通过类名直接调用类里面的方法的做法，我们就称之为：调用未绑定的父类方法
    • 直接调用未绑定的父类方法，是一种不太建议的做法，因为，有些情况下会造成：钻石继承的问题。
    class C:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def add(self):
        return self.x + self.y
    
    def mul(self):
        return self.x * self.y


class D(C):
    def __init__(self, x, y, z):
        # 调用父类C的__init__方法，初始化x和y属性, 也可以写为: C.__init__(x, y)
        super().__init__(x, y) # 这样写，我们就不用再去写 self.x = x, 我们直接用C类的构造函数
    # 将add 和 mul 方法也改写一下  
    def add(self):
        return C.add(self) + self.z # 直接调用类C的方法
    
    def mul(self):
        return C.mul(self) * self.z

    # 钻石继承

    class A:
    def __init__(self):
        print('hello I am A ~')
        
        
class B1(A):
    def __init__(self):
        # 这里我们先调用未绑定的父类A的 __init__() 方法
        A.__init__(self)
        print('hello I am B1 ~')

class B2(A):
    def __init__(self):
        # 这里我们先调用未绑定的父类A的 __init__() 方法
        A.__init__(self)
        print('hello I am B2 ~')

# 类C的话，同时继承了B1 和 B2
class C(B1, B2):
    def __init__(self):
        # 这里我们先调用未绑定的父类B1 和 B2 的 __init__() 方法
        B1.__init__(self)
        B2.__init__(self)
        print('hello I am C ~')

c1 = C() # hello I am A ~, 会调用两次，可以去联想js 里面，构造函数继承的几种方式里，也是多次调用
# hello I am A ~
# hello I am B1 ~
# hello I am A ~
# hello I am B2 ~
# hello I am C ~
    • 类C, 就是达到了钻石继承的条件了
      • 类C是同时继承类B1 和 类B2 的, 类B1和类B2 又是继承自类A的
      • 所以当我们类C，去调用类B1和类B2的构造函数的时候，类A的构造函数也会被跟着调用两次, B1 一次， B2一次

    钻石继承详解

    1. 钻石继承，我们只希望调用一次，程序的话，要精准, 不能说重复调用2次，这种是不严谨的
    2. 解决办法，就是调用super()函数
    3. super()函数，能够在父类中搜索指定的方法，并自动绑定好self 参数
    4. 类A，不用改
    5. 直接修改B1 和 B2, B1 里面，就不通过这个未绑定的父类的方法来调用了, 直接调用super() 函数就好了 A.init(self) 这里 修改为使用: super().init(self), 因为上面有写清继承关系 因为B1 是继承自A的，所以它会自动去搜寻父类，super().init(self) 这里面的 self 也不用显示的传进去了 super().init(), super() 函数，会帮你解决这个问题。
    • 使用super() 函数，类C里面，也不用再写 B1.init(self) 和 B2.init(self) 了， 直接写super().__init__(), 它自己就会去遍历，继承了哪些父类
    • 修改了之后，再来实例化c, 就不会重复了
    • 也就是说，只要使用 super() 函数去查找父类的方法，它就会自动按照这个MRO顺序，去搜索父类的相关方法，并且自动避免重复调用的问题。

    # MRO 顺序是什么？

    • 多继承里面，如果出现同名的属性或方法, Python会有一个明确的一个查找覆盖顺序, 这个顺序有一个官方术语叫做MRO
    • 英文全称: Method Resolution Order, 译为: 方法解析顺序

    要查找一个类的MRO有两种方法

    1. mro()方法,
    2. mro属性, 也就是：__mro__
    class A:
    def getName(self, name):
        self.name = name
class B:
    def getName(self, age):
        self.age = age

class C(A, B):
    pass

c = C()

print(C.mro()) # [<class '__main__.C'>, <class '__main__.A'>, <class '__main__.B'>, <class 'object'>]

# 之所以会出现object，是因为 object 是所有类的基类，它也会被隐式的继承
    • mro()
    class A:
    def __init__(self):
        print('hello I am A ~')

    class Displayer:
    def display(self, message):
        print(message)
    
class LoggerMixin:
    def log(self, message, filename='logfile.txt'):
        # 以追加模式打开文件：
        # 这个方法的功能，就是创建一个文件，并将这个message追加保存到文件中
        with open(filename, "a") as f:
            f.write(message)
    
    def display(self, message):
        super().display(message)
        self.log(message)
    
    
class MySubClass(LoggerMixin, Displayer):
    def log(self, message):
        super().log(message, filename='logfile.txt')
        
subclass = MySubClass()

subclass.display("First append.")

    对这个例子的解析，这个例子很值得研究，很有意思

    1. 首先呢，类MySubClass()，初始化了一个实例对象: subclass
    2. 然后，subclass 调用display()方法, 参数是："First append."
    3. 那么，这个调用的display()方法是谁的，首先不可能是自己的，因为类MySubClass()并没有实现这个方法
    4. 所以我们要去从父类里面(LoggerMixin, Displayer)查找，它的父类有2个，一个是：LoggerMixin类， 一个是: Displayer类
    5. 根据先左后右的原则，Python会去类LoggerMixin中去查找这个display()方法
    6. 找到了之后，我们发现，类LoggerMixin里面的这个display()方法, 是带了一个message参数，且看，display()方法实现了第一句就是super().display(message)，super()函数呢，就会去父类里边查找对应的方法是不是
    7. 那么这个类LoggerMixin的父类是谁呢? 它没有写父类，那默认就是 object类，任何类它都是object的一个子类
    8. 但是object呢，但是object作为一个总的基类, 是不可能会有display()这种方法的，那怎么办呢
    9. 其实这里就涉及到 MRO 顺序的问题，我们来看下 类MySubClass 的MRO 顺序
    10. (<class '__main__.MySubClass'>, <class '__main__.LoggerMixin'>, <class '__main__.Displayer'>, <class 'object'>)
    11. 因为super()函数是严重依赖MRO顺序的，最后才去找 object, 一层层往上找到的
    12. 所以，在LoggerMixin里面，display()方法调用super() 函数，它会先去类Displayer中查找，而不是object
    13. 所以，LoggerMixin类里面的display()方法调用super() 函数, 其实就是去调用类Displayer里面的display()方法
    14. 接下来，就是：self.log(message), 新的问题，这个log() 方法是谁的呢，这个方法呢，取决于self
    15. self 其实就是subclass这个对象本身, 之前学到的绑定，然后这个对象是来自于MySubClass的，
    16. 类MySubClass() 里面，刚好就有一个log()方法，那就是去执行类MySubClass() 里面的log()方法，仔细看，这个log() 方法里，又调用了super()函数
    17. super()函数呢，会自动去父类里面查找对应的方法， 根据MRO顺序，也就是：类LoggerMixin 里面的log() 方法
    18. 然后呢，类LoggerMixin 里面的log() 方法的功能就是：创建一个文件，并将这个message追加保存到文件中，还有个默认参数: filename

    如果看到类的名称中，包含Mixin，那么一般它是为了添加某个功能而后期插上去的，也就是混入，在不改变原有代码结构的基础上，增加一些额外的功能。

    # 私有变量__slots__

    所谓私有变量，就是指通过某种手段, 使得对象中的属性或方法无法被外部所访问。

    • 这其实是一种保护机制
    • 那种仅限从一个对象内部才能够访问的"私有变量"并不存在
    • 引入了一种叫做name mangling的一个机制, 译为: 名字改编、名称改写或者名称修饰
    • 语法：就是在名字的前面加上两个连续的下横线，
    1. 又定义一个set_x()来设置这个私有变量的值
    2. 然后再定义一个get_x()来获取这个私有变量的值
    3. 此时我们是无法直接通过变量名来访问到这个变量的, 既然说私有，那就是尽可能的避免你从外部去访问
    4. 想要访问__x 这个变量的值，就需要通过指定的接口
    5. 比如说这段代码里边的set_x()和get_x()的这两个方法
    class C:
    def __init__(self, x):
        self.__x = x # 前面两个下划线，代表私有变量
    
    def set_x(self, x):
        self.__x = x
    
    def get_x(self):
        print(self.__x)

# 初始化一个实例对象c

c = C(520)

# 此时我们没办法直接访问到__x 属性

# print(c.__x) # AttributeError: 'C' object has no attribute '__x'

# print(c.get_x()) # 520

# 通过set_x() 修改，私有属性的值

c.set_x(666)

print(c.get_x()) # 666

# 我们来看一下c的__dict__ 属性

print(c.__dict__) # {'_C__x': 666}

# 因此我们可以通过c._C__x 来访问 x 变量

print(c._C__x) # 666
    • 从上面的示例中，我们可以看出：下横线加上类名再加上变量的名字(eg: _C__x)，这就是名字改编术，python对私有变量的处理
    1. 所以呢，在Python中所谓的"私有变量"，就是把你想要私有的那个变量，程序底层偷偷给改了个名字
    2. 而且这个改名是有规律的，同样地，方法名也是同样的道理
    • eg: 我们这里要把方法给隐藏起来, 我们可以在它前面加上两个连续的下横线
    class D:

    • 因此，空间也就节约了出来，不要小看这些改动点

    • 使用了__slots__属性就没办法再拥有动态添加属性的功能了
    • 但是实际开发里，却又可以利用这个副作用来限制类属性的滥用

    反思：知道的越多，方法越多，要多学习技术点，提升技能。

    # 需要注意

    • 继承自父类的__slots__属性是不会在子类中生效的
    • Python只会关注各个具体的类中定义的__slots__属性
    • 但是这个__slots__属性是类C的
    class C:
    __slots__ = ["x", "y"]
    def __init__(self, x):
        self.x = x


# 类C里面，使用了__slots__属性

# 定义一个类E, 继承C

class E(C):
    pass

# 实例化一个对象
e = E(660)
e.y = 520

print(e.x)
print(e.y)

# 因为E是继承自C的，所以C能做的e也能做
# 然后我们看一下，动态的添加一个slots，因为C里面，我们这个类C的__slots__属性里，设置了 x 和 y
# 那我们给e动态的添加一个z, 

e.z = 999 # 也是可以的

# 我们再看一下e.__slots__
print(e.__slots__) # ['x', 'y']
# 虽然e有__slots__，但是这个__slots__是类C的， 这个是继承下来的
# 但它同时呢也会拥有一个__dict__属性

# 我们新添加的'z': 999就放到了这个__dict__属性里面
print(e.__dict__) # {'z': 999}, 且__dict__属性里，只会存放z, 并不会存放 x 和 y
    • 因此，继承自父类的__slots__属性, 是不会在子类中生效的, 子类依旧可以动态地添加属性
    • Python只关注 各个具体类中定义的__slots__ 属性
