    1. 数字（Number）
    2. 字符串（String）

    5. 字典（Dictionary）

    list = ['123', 46, 'str'];

    元组（Tuple） tuple = ('123', 46, 'str');

    字典（Dictionary） dict = {'name': '张三', 'age': 18};

    元组的元素不能修改

    List 和 元组 都是 里面的元素类型可以不一样

    字典：它是一个无序的 键(key) : 值(value) 的集合。

    # 1. Python 中的序列切片语法

    截取列表（或序列）的最后 N 个元素

    history = history[-MAX_HISTORY:] 的核心作用是强制将 history 截断为最后 5 个元素，确保其长度最多为 5，

    • 常用于限制历史记录（如对话、操作日志、数据缓存）的长度。

    # 场景 1：history 长度 > 6（截断）

    MAX_HISTORY = 6
history = [1, 2, 3, 4, 5, 6, 7, 8]  # 长度为8
history = history[-MAX_HISTORY:]
print(history)  # 输出：[3, 4, 5, 6, 7, 8]（只保留最后6个）

    # 场景 2：history 长度 = 6（无变化）

    MAX_HISTORY = 6
history = [1, 2, 3, 4, 5, 6]  # 长度为6
history = history[-MAX_HISTORY:]
print(history)  # 输出：[1, 2, 3, 4, 5, 6]（原样保留）

    # 场景 3：history 长度 < 6（保留全部）

    MAX_HISTORY = 6
history = [1, 2, 3]  # 长度为3
history = history[-MAX_HISTORY:]
print(history)  # 输出：[1, 2, 3]（不足6个时，不截断，保留所有）

    # 核心用途

    这行代码是限制历史记录长度的“惯用写法”，比如：

    • 聊天机器人的对话历史，只保留最近 5 轮；
    • 传感器数据缓存，只存最近 5 条采集值；
    • 操作日志，只保留最近 5 次操作记录。

    # 补充说明

    • 若 history 是空序列（如 []），执行后仍为空，不会报错；
    • 该语法对字符串、元组等可切片对象同样生效（元组执行后仍为元组，列表仍为列表）。

    # 核心用途

    这行代码常用于“分组处理”场景，比如：

    1. 对话历史分组：将连续的聊天记录按“一问一答（2 条为一组）”拆分；
    2. 数据成对处理：传感器采集的成对数据（如温度+湿度）分组；
    3. 批量处理：按固定数量（2 个）拆分列表，方便后续批量操作。

    # 补充注意

    • 切片 history[i:i+2] 不会报错：即使 i+2 超出列表长度，也只会取到最后一个元素；
    • 步长和分组大小可灵活调整：比如把 2 改成 3，就是每 3 个元素为一组。

    # 2. isinstance

    isinstance() 是判断一个对象是否属于指定类型（或多个类型之一） 的内置函数，核心作用是做「类型检查」—— 确保变量的类型符合预期，避免因类型错误导致程序崩溃。

    # 一、isinstance() 基本语法

    isinstance(对象, 类型/类型元组)

if isinstance(result, list):
    • 返回值：布尔值（True 或 False）
      • 若「对象」是「指定类型」（或多个类型之一），返回 True；
      • 否则返回 False。

    isinstance(result, list):的作用就是「判断变量result` 是否为列表类型」, 核心价值是类型校验—— 确保后续代码（如遍历、索引取值）能安全执行，避免因类型不匹配导致的程序错误，尤其适合处理 API 返回、文件读取等不确定类型的场景。

    # 4. sys.argv[1]

    import sys

sys.argv

    在 Python 中，sys.argv 是用于获取命令行参数的核心工具，相关属性的作用和使用场景如下，结合实例说明更易理解：

    # 1. 先明确核心背景

    当你在 命令行（终端/CMD） 中运行 Python 脚本时，除了脚本文件名本身，还可以在后面跟额外的“参数”，这些参数会被 Python 捕获并存储在 sys.argv 中，供脚本内部使用。

    比如在命令行执行：

    python test.py 张三 20 95.5

    这里：

    • python 是解释器命令
    • test.py 是脚本文件名
    • 张三、20、95.5 是额外传入的命令行参数 sys.argv 会把这些信息统一存储起来。

    在命令行执行：

    python test.py 张三 20 95.5

    # 输出结果：

    sys.argv 完整列表： ['test.py', '张三', '20', '95.5']
脚本文件名： test.py
额外参数个数： 3
第一个额外参数（sys.argv[1]）： 张三
第二个额外参数（sys.argv[2]）： 20  转换为整数： 20
第三个额外参数（sys.argv[3]）： 95.5  转换为浮点数： 95.5

    总结：sys.argv 是 Python 脚本与命令行交互的基础，核心作用是“接收外部传入的参数”，sys.argv[1:] （切片）可直接获取所有额外参数。

    # Flask 框架

    from flask import Flask, jsonify, make_response
from flask_socketio import SocketIO, emit

socketio = SocketIO(cors_allowed_origins='*', async_mode='threading')
app = Flask(__name__)
app.config["JSON_AS_ASCII"] = False
socketio.init_app(app)

    基于 Flask 框架搭建 Web 应用+WebSocket 实时通信服务 的核心初始化代码，主要作用是创建 Flask 应用实例、配置基础参数，并初始化支持跨域的 WebSocket 服务，让应用既能处理普通 HTTP 请求，又能实现客户端与服务器的实时双向通信。

    # 1. 导入依赖库

    from flask import Flask, jsonify, make_response
from flask_socketio import SocketIO, emit
    • Flask：创建 Web 应用的核心类（Flask 框架的入口）；
    • jsonify：快速将 Python 字典/数据转换成 JSON 格式的 HTTP 响应（方便接口返回 JSON 数据）；
    • make_response：自定义 HTTP 响应（比如设置响应头、状态码等）；
    • SocketIO：Flask 生态中支持 WebSocket 协议的扩展（负责实时通信）；
    • emit：WebSocket 中用于发送消息的函数（服务器向客户端发消息，或客户端向服务器发消息）。

    # 2. 初始化 WebSocket 实例

    socketio = SocketIO(cors_allowed_origins='*', async_mode='threading')

    创建 WebSocket 服务实例，并配置两个关键参数：

    • cors_allowed_origins='*'：允许所有域名的客户端跨域连接（比如你的前端页面部署在 http://localhost:8080，后端 Flask 部署在 http://localhost:5000，跨域请求不会被浏览器拦截，开发阶段常用 * 简化配置，生产环境建议指定具体域名）；
    • async_mode='threading'：指定 WebSocket 的异步模式为“线程模式”（Flask 原生是同步框架，通过线程处理并发的 WebSocket 连接，保证实时通信不阻塞其他请求，适合中小型应用）。

    # 3. 初始化 Flask 应用实例

    app = Flask(__name__)
    • Flask(__name__)：创建 Flask 应用核心对象 app；
    • __name__ 是 Python 的内置变量，代表当前模块名，Flask 会通过它定位应用的根目录（比如查找静态文件、模板文件的路径）。

    # 4. 配置 Flask 应用参数

    app.config["JSON_AS_ASCII"] = False
    • 关闭 JSON 响应的 ASCII 编码限制（默认情况下，jsonify 会把中文等非 ASCII 字符转换成 \uXXXX 格式，设置为 False 后，会直接返回中文原文，方便前端显示）。

    # 5. 关联 WebSocket 与 Flask 应用

    socketio.init_app(app)
    • 将之前创建的 WebSocket 实例 socketio 与 Flask 应用 app 绑定，让 Flask 应用同时支持：
      1. 普通 HTTP 接口（比如 @app.route('/api/data') 定义的接口）；
      2. WebSocket 实时通信（比如客户端通过 ws://xxx 连接服务器，实现消息推送、实时聊天等）。

    # 这段代码的典型使用场景

    有了这些初始化后，你可以：

    1. 写普通 HTTP 接口（比如查询数据）：
      @app.route('/api/get_user', methods=['GET'])
def get_user():
    return jsonify({"name": "张三", "age": 20})  # 中文正常显示
    2. 写 WebSocket 实时通信逻辑（比如服务器向客户端推送消息、客户端之间聊天）：
      # 客户端连接 WebSocket 时触发
@socketio.on('connect')
def handle_connect():
    print("有客户端连接成功！")
    emit('welcome', '欢迎加入实时通信！')  # 向连接的客户端发消息

# 接收客户端发送的"chat_message"类型消息
@socketio.on('chat_message')
def handle_chat_message(msg):
    print(f"收到消息：{msg}")
    socketio.emit('broadcast_message', msg)  # 向所有连接的客户端广播消息

    # 总结

    这段代码是一个 支持跨域、支持 JSON 中文显示、同时具备 HTTP 接口和 WebSocket 实时通信能力 的 Flask 应用基础骨架。

    后续只需：

    • 用 @app.route 定义 HTTP 接口；
    • 用 @socketio.on 定义 WebSocket 消息处理逻辑；
    • 运行 socketio.run(app, debug=True) 启动服务，即可实现普通接口调用和实时消息交互（比如聊天、实时数据推送等场景）。
