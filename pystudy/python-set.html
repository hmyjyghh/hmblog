    # 集合

    # 创建集合和字典，

    • eg: 不可变的类型，才可以作为集合的键

    可哈希, 不可变的类型(字符串、元组)、元组也是一个不可变的容器

    • hash(object): 获取一个对象的哈希值

    • 对于字典和集合的实现来说，

    后续研究方向: 字典和集合高效背后的玄机。

    # 元组，它也是一个不可变的容器，也是可以计算到一个哈希值的

    print(hash("Hmyjy"), "字符串") # 6524292841515017885 字符串
print(hash((1,2,3)), "元组") # 529344067295497451 元组

# 列表是不可哈希的，它不能作为字典的键
print(hash([1,2,3])) # 会报错：TypeError: unhashable type: 'list'

    # 只有可哈希的对象，才有资格作为字典的键

    print({"Python": 520, "Hmyjy": 1314}) # {'Python': 520, 'Hmyjy': 1314}


print({[1,2,3]:"Hmyjy"}) # TypeError: unhashable type: 'list'

    # 特别注意！！！

    # 这个是字典
{"Python": 520, "Hmyjy": 1314}

# 这里是：集合，非键值对形式
{"Python", "Hmyjy", 520, 1314}

# 集合报错写法:
{"Python", "Hmyjy", 520, 1314, [6,6,6]}

print({"Python", "Hmyjy", 520, 1314, [6,6,6]}) # TypeError: unhashable type: 'list'
    • 和集合的元素(存在疑惑，待确认)

    # 实现一个嵌套的集合

    • 集合是一个可变的容器, 而可变的容器是不可哈希的
    • 不可哈希，则不能作为字典的键
    x = {1,2,3}
y = {x,4,5}

print(y) # TypeError: unhashable type: 'set'

# 如果想实现嵌套，可以使用`frozenset`
# 因为使用了frozenset， 就成了不可变的，这样就实现了一个嵌套的集合
x = frozenset({1,2,3})
y = {x,4,5}

print(y) # {frozenset({1, 2, 3}), 4, 5}

    frozenset：存在即合理

    # 集合对象，也包含了很多的内置方法

    # 优化点

    • 添加一行代码, 提高10000倍以上的执行效率
    • 下面的代码，很耗时，不要随便执行，要注意
    import random
import timeit

haystack = [random.randint(1, 10000000) for i in range(10000000)]
haystack = set(haystack) # 添加的一行，优化代码
needles = [random.randint(1, 1000) for i in range(1000)]

# 请在此处添加一行代码，使得查找过程的执行效率提高 10000 倍以上。

def find():  
    found = 0  
    for each in needles:  
        if each in haystack:  
            found += 1  

    print(f"一共找到{found}个匹配。")

t = timeit.timeit("find()", setup="from __main__ import find", number=1)
print(f"查找过程一共消耗{t}秒。")

# 一共找到624个匹配。
# 查找过程一共消耗0.0007713001687079668秒。

    为什么列表变成集合之后, 效率竟然提高了这么多呢?
其实是因为集合的背后是有散列表的支持, 而列表则没有!
所以, 列表的每次查找都需要从头到尾地扫描, 而集合则只需要简单地查表操作
打个比方, 这就好比你在新华字典里面直找一个"龟"字, in在列表上的查找方法是从头翻到尾, 挨个对比是不是"龟"字;
而in在集合上的查找方式是通过笔画的索引, 直接就拿到了这个"龟"字所在的页码。

    # 集合高效的原因

    集合背后，就是有散列表的支持

    • 代价就是要牺牲海量的存储空间

    # 有关集合

    1. 集合是无序的，所以不能通过下标索引的形式去获取
    s = set('Hmyjy')

s[0] # TypeError: 'set' object is not subscriptable(类型错误：'set'对象不可下标)
    1. 判断某个元素是否存在于集合中
    • 集合的特性：唯一性
    • 无顺序
    • 可以遍历
    s = set('Hmyjy')

print('H' in s) # True


for e in s:
    print(e) # 这个示例很巧妙，1. 验证了唯一性，无重复，2. 无顺序， 3. 集合也可以遍历
# H
# y
# m
# j
    • 利用这个唯一性的特性，我们可以很方便的实现去重的操作
    s = set([1,1,2,3,6,7,8,9,3,2])

print(s) # {1, 2, 3, 6, 7, 8, 9}

    # 在实际的开发中

    • 可以很方便的判断，一个列表中是否存在相同的元素
    len(s) == len(set(s))

# set(s) 就会把它变成一个去重之后的结果

    # 集合推导式

    • 用于创建集合（自动去重），语法与列表推导式类似，但用{}包裹。
    • 语法：{表达式 for 变量 in 可迭代对象 if 条件}
    obj = {s for s in "Hmyjy"}

print(obj) # {'j', 'y', 'H', 'm'}, 符合集合无序这个特征

    # 集合的各种方法大合集

    以下是提取的图中信息（Markdown格式）：

    方法 说明
    s.copy() 返回s集合的一个浅拷贝
    s.isdisjoint(other) 如果s集合中没有与other容器存在共同的元素，那么返回True，否则返回False
    s.issubset(other) 如果s集合是other容器的子集（注1），那么返回True，否则返回False
    s.issuperset(other) 如果s集合是other容器的超集（注2），那么返回True，否则返回False
    s.union(*others) 返回一个新集合，其内容是s集合与others容器的并集（注3）
    s.intersection(*others) 返回一个新集合，其内容是s集合与others容器的交集（注4）
    s.difference(*others) 返回一个新集合，其内容是存在于s集合中，但不存在于others容器中的元素（注5）
    s.symmetric_difference(other) 返回一个新集合，其内容是排除掉s集合和other容器中共有的元素后，剩余的所有元素

    # s.copy()

    # 定义原集合
s = {1, 2, 3}
# 浅拷贝原集合
s_copy = s.copy()

# 验证拷贝结果
print("原集合 s:", s)          # 输出: 原集合 s: {1, 2, 3}
print("拷贝后的集合 s_copy:", s_copy)  # 输出: 拷贝后的集合 s_copy: {1, 2, 3}

# 修改原集合，观察拷贝集合是否受影响
s.add(4)
print("修改后的 s:", s)       # 输出: 修改后的 s: {1, 2, 3, 4}
print("s_copy 不受影响:", s_copy)  # 输出: s_copy 不受影响: {1, 2, 3}

    # s.union(*others) 示例

    # 定义基础集合
s1 = {1, 2, 3}
s2 = {3, 4, 5}
s3 = {5, 6, 7}

# 求 s1 和 s2 的并集
union1 = s1.union(s2)
print("s1 和 s2 的并集:", union1)  # 输出: s1 和 s2 的并集: {1, 2, 3, 4, 5}

# 求 s1、s2、s3 多个集合的并集
union2 = s1.union(s2, s3)
print("s1、s2、s3 的并集:", union2)  # 输出: s1、s2、s3 的并集: {1, 2, 3, 4, 5, 6, 7}
