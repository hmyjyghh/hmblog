    import re

    # jieba==0.42.1

    jieba 是 Python 中一款非常流行的中文分词库，版本 0.42.1 是其历史版本之一，主要作用是将连续的中文文本按照语义拆分成一个个独立的词语（分词），是中文自然语言处理（NLP）中最基础且常用的工具之一。

    具体来说，它的核心功能包括：

    1. 中文分词：支持精确模式（最常用，试图将句子最精确地切开）、全模式（把句子中所有可能的词语都扫描出来）、搜索引擎模式（在精确模式基础上，对长词再次切分，适合搜索引擎分词）。
    2. 自定义词典：允许用户添加自定义词语，以解决专有名词（如人名、地名、专业术语）无法被正确识别的问题。
    3. 词性标注：可以为分词后的词语标注词性（如名词、动词、形容词等）。
    4. 并行分词：支持利用多线程加速分词过程，提升处理大量文本时的效率。

    由于中文文本不像英文那样有天然的空格分隔词语，分词是中文文本处理的前提步骤，因此 jieba 广泛应用于文本挖掘、情感分析、信息检索等场景。

    # 线程池

    with concurrent.futures.ThreadPoolExecutor(max_workers=MAX_WORKERS) as executor:
    futures = {doc.metadata['unique_id']: executor.submit(chat, LLM_CLEAN_PROMPT.format(doc.page_content)) for doc in groups}

    在这段代码中，concurrent.futures.ThreadPoolExecutor 是 Python 标准库中用于创建线程池的工具，其核心作用是高效地并发执行多个任务，主要体现在以下几个方面：

    # 1. 管理线程资源，避免频繁创建/销毁线程的开销

    线程的创建和销毁需要消耗系统资源（如 CPU、内存）。如果为每个任务单独创建一个线程，当任务数量较多时，频繁的线程创建/销毁会显著降低效率。
    ThreadPoolExecutor 会预先创建指定数量的线程（通过 max_workers 控制），并复用这些线程执行多个任务，减少资源消耗。

    # 2. 并发执行任务，提高程序运行效率

    在这段代码中，executor.submit(chat, ...) 会将 chat 函数（可能是一个耗时操作，如调用 LLM 接口、网络请求等）提交到线程池。
    线程池会自动分配空闲线程执行这些任务，实现多个 chat 任务的并发执行（注意：Python 由于 GIL 限制，CPU 密集型任务并发效果有限，但 I/O 密集型任务如网络请求、文件读写等能显著提升效率）。

    # 3. 简化并发编程逻辑

    ThreadPoolExecutor 封装了线程管理的细节（如线程创建、任务分配、结果回收等），无需手动操作 threading 模块的线程对象。
    通过 futures 字典可以追踪每个任务（通过 unique_id 关联），后续可通过 future.result() 获取任务返回值，或处理异常。

    # 总结

    在这段代码中，ThreadPoolExecutor 的作用是：通过线程池并发执行多个 chat 任务，减少线程管理开销，利用并发提升处理效率（尤其适合 I/O 密集型的 LLM 调用场景），同时简化并发逻辑的实现。

    # from more_itertools import divide

    from more_itertools import divide 中的 divide 函数用于将一个可迭代对象（如列表、元组等）均匀地分割成指定数量的子序列，返回一个包含这些子序列的迭代器。

    # 核心作用：

    将原始可迭代对象拆分为 n 个部分，尽可能让每个部分的元素数量相近（若总元素数不能被 n 整除，前几个子序列会多1个元素）。

    # 示例：

    from more_itertools import divide

data = [1, 2, 3, 4, 5, 6, 7]

# 分成3个子序列
parts = divide(3, data)
print(list(parts))  # 输出：[[1, 2, 3], [4, 5], [6, 7]]
    • 总元素7个，分成3份：前1份3个元素，后2份各2个元素，尽可能均匀。

    # 与普通分割的区别：

    普通切片（如 data[i::n]）是按“间隔取元素”分割，而 divide 是按“连续片段”分割，更符合“均匀分块”的需求。

    常用于需要将任务/数据平均分配给多个 worker（如多线程/多进程处理）的场景。
