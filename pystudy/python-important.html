    import re

    # jieba==0.42.1

    jieba 是 Python 中一款非常流行的中文分词库，版本 0.42.1 是其历史版本之一，主要作用是将连续的中文文本按照语义拆分成一个个独立的词语（分词），是中文自然语言处理（NLP）中最基础且常用的工具之一。

    具体来说，它的核心功能包括：

    1. 中文分词：支持精确模式（最常用，试图将句子最精确地切开）、全模式（把句子中所有可能的词语都扫描出来）、搜索引擎模式（在精确模式基础上，对长词再次切分，适合搜索引擎分词）。
    2. 自定义词典：允许用户添加自定义词语，以解决专有名词（如人名、地名、专业术语）无法被正确识别的问题。
    3. 词性标注：可以为分词后的词语标注词性（如名词、动词、形容词等）。
    4. 并行分词：支持利用多线程加速分词过程，提升处理大量文本时的效率。

    由于中文文本不像英文那样有天然的空格分隔词语，分词是中文文本处理的前提步骤，因此 jieba 广泛应用于文本挖掘、情感分析、信息检索等场景。

    # 线程池

    with concurrent.futures.ThreadPoolExecutor(max_workers=MAX_WORKERS) as executor:
    futures = {doc.metadata['unique_id']: executor.submit(chat, LLM_CLEAN_PROMPT.format(doc.page_content)) for doc in groups}

    在这段代码中，concurrent.futures.ThreadPoolExecutor 是 Python 标准库中用于创建线程池的工具，其核心作用是高效地并发执行多个任务，主要体现在以下几个方面：

    # 1. 管理线程资源，避免频繁创建/销毁线程的开销

    线程的创建和销毁需要消耗系统资源（如 CPU、内存）。如果为每个任务单独创建一个线程，当任务数量较多时，频繁的线程创建/销毁会显著降低效率。
    ThreadPoolExecutor 会预先创建指定数量的线程（通过 max_workers 控制），并复用这些线程执行多个任务，减少资源消耗。

    # 2. 并发执行任务，提高程序运行效率

    在这段代码中，executor.submit(chat, ...) 会将 chat 函数（可能是一个耗时操作，如调用 LLM 接口、网络请求等）提交到线程池。
    线程池会自动分配空闲线程执行这些任务，实现多个 chat 任务的并发执行（注意：Python 由于 GIL 限制，CPU 密集型任务并发效果有限，但 I/O 密集型任务如网络请求、文件读写等能显著提升效率）。

    # 3. 简化并发编程逻辑

    ThreadPoolExecutor 封装了线程管理的细节（如线程创建、任务分配、结果回收等），无需手动操作 threading 模块的线程对象。
    通过 futures 字典可以追踪每个任务（通过 unique_id 关联），后续可通过 future.result() 获取任务返回值，或处理异常。

    # 总结

    在这段代码中，ThreadPoolExecutor 的作用是：通过线程池并发执行多个 chat 任务，减少线程管理开销，利用并发提升处理效率（尤其适合 I/O 密集型的 LLM 调用场景），同时简化并发逻辑的实现。

    # from more_itertools import divide

    from more_itertools import divide 中的 divide 函数用于将一个可迭代对象（如列表、元组等）均匀地分割成指定数量的子序列，返回一个包含这些子序列的迭代器。

    # 核心作用：

    将原始可迭代对象拆分为 n 个部分，尽可能让每个部分的元素数量相近（若总元素数不能被 n 整除，前几个子序列会多1个元素）。

    # 示例：

    from more_itertools import divide

data = [1, 2, 3, 4, 5, 6, 7]

# 分成3个子序列
parts = divide(3, data)
print(list(parts))  # 输出：[[1, 2, 3], [4, 5], [6, 7]]
    • 总元素7个，分成3份：前1份3个元素，后2份各2个元素，尽可能均匀。

    # 与普通分割的区别：

    普通切片（如 data[i::n]）是按“间隔取元素”分割，而 divide 是按“连续片段”分割，更符合“均匀分块”的需求。

    常用于需要将任务/数据平均分配给多个 worker（如多线程/多进程处理）的场景。

    # 文档解析

    # PyMuPDF，别名：fitz

    PyMuPDF 是处理 PDF 的轻量级高性能库，其核心优势, 在于能完美匹配布局感知分块的需求

    • PyMuPDF 能提取表格坐标，但复杂表格（合并单元格、跨页）需专用库。

    # 1. 精准的布局元素解析能力

    PyMuPDF 能直接提取PDF中每个文本/图形元素的几何坐标（x/y坐标、宽高、页码）、字体属性（大小、粗细、颜色）、文本行/块归属，这些是“布局感知”的核心数据：

    • 例：通过 page.get_text("dict") 可获取结构化文本数据，包含每个文本块的bbox（边界框）、lines（行）、spans（字符段），能直接区分“标题（大字体、粗体、顶部位置）”“正文（常规字体、连续文本块）”“页眉页脚（固定坐标区域）”等；
    • 支持提取表格的单元格坐标、图片的位置和内容，为识别“表格/图表”等布局元素提供基础。

    # 2. 轻量化+高性能

    相比PDFMiner、pdfplumber等纯Python库，PyMuPDF基于C++实现，解析速度快（处理百页PDF仅秒级），内存占用低，适合批量文档处理的业务场景（如合同解析、财报分析、档案数字化）。

    # 3. 灵活的内容提取与格式保留

    • 可按布局区域提取文本（如只提取正文区域、过滤页眉页脚）；
    • 支持保留文本的排版格式（如换行、缩进），甚至可提取带格式的HTML/Markdown，贴合“保留原始布局结构”的需求。

    # 二、实际业务中用PyMuPDF做布局感知分块的核心流程

    import fitz  # PyMuPDF

def layout_aware_chunking(pdf_path):
    doc = fitz.open(pdf_path)
    chunks = []  # 存储分块结果，每个块包含类型、内容、坐标、页码
    for page_num, page in enumerate(doc):
        # 1. 提取结构化文本（含布局信息）
        text_dict = page.get_text("dict")
        for block in text_dict["blocks"]:
            if block["type"] == 0:  # 文本块（排除图片/图形块）
                # 2. 分析布局特征（坐标、字体、文本长度）
                bbox = block["bbox"]  # (x0, y0, x1, y1) 左上角/右下角坐标
                first_span = block["lines"][0]["spans"][0]
                font_size = first_span["size"]
                font_bold = first_span["flags"] & 2 ** 4  # 粗体标识
                text = "".join([span["text"] for line in block["lines"] for span in line["spans"]]).strip()
                
                # 3. 布局元素分类（核心：根据特征判断块类型）
                if font_size > 16 and font_bold:
                    chunk_type = "标题"
                elif font_size > 12 and font_bold:
                    chunk_type = "子标题"
                elif bbox[1] < 50:  # 页面顶部50px内
                    chunk_type = "页眉"
                elif bbox[3] > page.rect.height - 50:  # 页面底部50px内
                    chunk_type = "页脚"
                elif len(text) > 100:
                    chunk_type = "正文文本"
                else:
                    chunk_type = "其他"
                
                chunks.append({
                    "page": page_num + 1,
                    "type": chunk_type,
                    "content": text,
                    "bbox": bbox
                })
    return chunks

# 调用示例
pdf_chunks = layout_aware_chunking("复杂文档.pdf")
for chunk in pdf_chunks[:5]:  # 打印前5个分块
    print(f"页码：{chunk['page']} | 类型：{chunk['type']} | 内容：{chunk['content'][:50]}...")

    # 三、复杂场景

    PyMuPDF 擅长“原生PDF（可复制文本）”的布局解析，但面对以下场景需结合其他工具：

    # 1. 扫描件/图片型PDF（无文本）

    需先做OCR提取文本+布局信息，常用组合：

    • PyMuPDF 提取图片 → Tesseract/OCRmyPDF 做OCR（保留坐标） → 再用PyMuPDF整合布局；
    • 进阶：阿里云/百度OCR的“版式分析”接口（直接返回标题、正文、表格等布局元素）。

    # 2. 高精度表格/复杂图表解析

    PyMuPDF 能提取表格坐标，但复杂表格（合并单元格、跨页）需专用库：

    • 表格：pdfplumber（基于坐标的表格解析）、Camelot；
    • 图表：先截图（PyMuPDF）→ 用Matplotlib/PIL分析，或调用GPT-4V识别图表内容。

    # 3. 多格式文档（Word/Excel/PowerPoint）

    先转换为PDF（python-docx/xlwings + PyMuPDF），再做布局分块；或直接用：

    • python-docx 解析Word的原生布局（标题级别、段落样式、表格）；
    • pdf2docx 将PDF转为Word后解析布局（适合需保留格式的场景）。

    # 4. 企业级布局理解（端到端模型）

    若需“全自动识别所有布局元素”（无需手动规则），需结合布局理解模型：

    • 开源模型：LayoutLMv3（微软，支持中文，输入文本+坐标+图像，输出布局元素类型）；
    • 商用模型：阿里云DocMind、腾讯文智的版式分析API，直接返回结构化的布局分块结果。

    # 四、总结

    1. 核心结论：布局感知分块的业务场景中，PyMuPDF 是基础且核心的工具，尤其适合可复制文本的PDF，能快速获取布局坐标和文本信息，支撑分块规则的实现；
    2. 场景延伸：
      • 简单场景（原生PDF、常规布局）：仅用PyMuPDF即可完成布局分块；
      • 复杂场景（扫描件、复杂表格、多格式）：PyMuPDF + OCR/专用解析库/布局理解模型；
      • 企业级场景：直接调用布局理解模型（LayoutLM/商用API），PyMuPDF 作为数据预处理工具。
