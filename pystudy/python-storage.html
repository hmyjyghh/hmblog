    永久存储

    寒梦

    # 永久存储

    1. 当我们程序运行起来的时候，大多数数据都是从硬盘转移到了内存中，内存跟我们CPU之间的数据传输速度，要比这个硬盘跟CPU之间要快上很多倍

    # 打开文件

    open() 函数

    • open() 函数所返回的文件对象类型取决于所使用的模式：
      • 当使用 open() 以文本模式（'w'，'r'，'wt'，'rt' 等）打开文件时，它将返回 io.TextIOBase（准确的说是 io.TextIOWrapper）的一个子类
      • 当使用缓冲以二进制模式打开文件时，返回的类是 io.BufferedIOBase 的一个子类，具体的类会有多种：在只读的二进制模式下，它将返回 io.BufferedReader；在写入二进制和追加二进制模式下，它将返回 io.BufferedWriter；而在读/写模式下，它将返回 io.BufferedRandom
      • 当禁止缓冲时，则会返回原始流，即 io.RawIOBase，io.FileIO的一个子类 。

    # usage

    open(file, mode='r', buffering=-1, encoding=None, errors=None, newline=None, closefd=True, opener=None)

    # file、mode 参数

    参数 含义
    file 指定一个将要打开的文件的路径（绝对路径或相对路径）
    mode 可选参数，指定文件的打开模式：

    参数 含义

    - 'r'：读取（默认）
    - 'w'：写入（如果文件已存在则先截断清空文件）
    - 'x'：排他性创建文件（如果文件已存在则打开失败）
    - 'a'：追加（如果文件已存在则在末尾追加内容），注1
    - 'b'：二进制模式，注2
    - 't'：文本模式（默认），注3
    - '+'：更新文件（读取和写入）

    # 文件读取， 读取，写入

    • 注意打开一个文件(以写的模式打开)， 可能会把文件内容给清空，特别注意！！！
    • 以写入模式("w")，打开文件，直接关闭，会把内容清空
    f = open("hm.txt", "w")
f.close()
# 打开完之后，我不做任何事情，就把文件对象关闭，会把内容清掉
# mode="write", 这是文件操作系统，一定要注意的！！！

    # 文件对象的各种方法大合集

    方法 含义
    f.close() 关闭文件对象
    f.flush() 将文件对象中的缓存数据写入到文件中（不一定有效）
    f.read(size=-1, /) 从文件对象中读取指定数量的字符（或者遇到 EOF 停止）；当未指定该参数，或该参数为负值的时候，读取剩余的所有字符
    f.readable() 判断该文件对象是否支持读取（如果返回的值为 False，则调用 read() 方法会导致 OSError 异常）
    f.readline(size=-1, /) 从文件对象中读取一行字符串（包括换行符），如果指定了 size 参数，则表示读取 size 个字符
    f.readlines(size=-1, /) 从文件对象中读取所有字符串（包括换行符），然后按行为单位存储到列表中。如果指定了 size 参数，则表示读取 size 个字符（如果 size 参数指定的字符个数少于第一行字符个数，则仍然存放第一行字符，其他行也一样，它是按“行”为单位存储的）
    f.seek(offset, whence=0, /) 修改文件指针的位置，从 whence 参数指定的位置（0 代表文件起始位置，1 代表当前位置，2 代表文件末尾）偏移 offset 个字节，返回值是新的索引位置
    f.seekable() 判断该文件对象是否支持修改文件指针的位置（如果返回的值为 False，则调用 seek()、tell()、truncate() 方法都会导致 OSError 异常）
    f.tell() 返回当前文件指针在文件对象中的位置
    f.truncate(pos=None, /) 将文件对象截取到 pos 的位置，默认是截取到文件指针当前指定的位置
    f.write(text, /) 将字符串写入到文件对象中，并返回写入的字符数量（字符串的长度）
    f.writable() 判断该文件对象是否支持写入（如果返回的值为 False，则调用 write() 方法会导致 OSError 异常）
    f.writelines(lines, /) 将一系列字符串写入到文件对象中（不会自动添加换行符，所以通常是人为地加在每个字符串的末尾）

    有两种方法可以将字符串写入到文件对象中是不是

    1. write()
    2. writelines()
      • 这个writelines(), 它不会自动添加这个换行符
      • 所以，如果希望传入进去的多个字符串，带换行的话，需要显示指定\n
    f = open("hm.txt", "w") # 默认写入当前文件，所在根目录
f.write("I love Python hh.") # 会返回写入的字符数, 16
f.writelines(["I love Hmyjy.\n", "I love my wife."]) # 没有返回值


f = open("hm.txt", "r+") # r+ 的形式(也就是以更新文件的形式来打开)，既可以写入，也可以读取

print(f.writable()) # True 既可以写入

print(f.readable()) # True 也可以读取
    • 由于Python的文件呢是支持迭代的, 我们是可以将文件放到for语句里面去实现读取

for e in f:
    print(e)

# 截取
f.truncate(29) # 那么就是截取到文件指针当前指定的位置
# 输出: I love Python hh.I love Hmyjy

    # with语句和上下文管理器

    • with语句，也就是上下文管理器，它为文件操作提供了一种更为优雅的实现方式
    f = open("hm.txt", "w")
f.write("I love Hmyjy.\n I Love 520.")
f.close()

with open("fishC.txt", "w") as f:
    f.write("I love fishC.")

# 这里是特意注释掉的，使用with 语句，不需要再手动关闭文件
# f.close() # 注意，使用with 我们就不需要再去关闭文件了

    # 文件操作的三板斧

    1. 上文就是打开文件
    2. 往文件里写入东西
    3. 下文就是关闭文件
    • 这就是上下文管理器做的事情
    • 使用了上下文管理器，我们就不需要再去手动的关闭文件
    • 文件处理的代码只需要放在with语句的缩进里边，就可以了，这样的代码，显得更优雅
    • 使用上下文管理器最大的优势，是能够确保资源的释放(在这里，也就是文件的正常关闭了，我们不用去手动关闭文件了)

    # 使用with 语句的好处

    1. 任意新建一个.py 文件，写入
    f = open("hm.txt", "w")
f.write("I Love hmyjy.")
1 / 0 # 由于这一步报错，导致f.close() 函数未能执行，所以内容未能写入文件
# 如果代码报错，程序并没有机会执行到这个文件关闭的一个操作
f.close()

# 但是如果使用 with, 情况就会不一样
with open("fishC.txt", "w") as f:
    f.write("I love fishC.")
    1 / 0 # 虽然这一步报错了，但是with 上下文管理器，依然帮你确保这个文件可以正常的关闭

    # pickle

    pickle模块解决的就是一个，永久存储Python对象的问题

    • 它允许你将字符串、列表、字典, 这些Python对象给保存为文件的形式

    将python 对象(元组、列表、字典)序列化的一个操作

    • 所谓序列化，就是将Python对象转换为二进制字节流的这么一个过程

    • 就是将源代码，变成0101001的二进制组合

    • dump()

    • load()

    # dump()

    import pickle

x, y, z = 1, 2, 3
s = "Hmyjy" 
l = ["寒梦", 520, 3.14] 
d = {"one":1, "two":2}

# 后缀名为:.pkl 格式，写入模式，指定为二进制格式
with open("data.pkl", "wb") as f: 
    pickle.dump(x, f) 
    pickle.dump(y, f) 
    pickle.dump(z, f) 
    pickle.dump(s, f) 
    pickle.dump(l, f) 
    pickle.dump(d, f)

    # load()

    import pickle

with open("data.pkl", "rb") as f:
    x = pickle.load(f)
    y = pickle.load(f)
    z = pickle.load(f)
    s = pickle.load(f)
    l = pickle.load(f)
    d = pickle.load(f)

print(x, y, z, s, l, d, sep="\n") # 每输出一个值，加一个换行符
# 1
# 2
# 3
# Hmyjy
# ['寒梦', 520, 3.14]
# {'one': 1, 'two': 2}

    # 优化写法

    • 一个个写，太麻烦了，使用元组的形式，进行序列化
    import pickle

x, y, z = 1, 2, 3
s = "Hmyjy" 
l = ["寒梦", 520, 3.14] 
d = {"one":1, "two":2}

# 后缀名为:.pkl 格式，写入模式，指定为二进制格式
# 这里第一个参数传入一个元组的形式这样子
with open("data.pkl", "wb") as f: 
    pickle.dump((x,y,z,s,l,d), f)


# 读取，解包
with open("data.pkl", "rb") as f: 
    x, y, z, s, l, d = pickle.load(f) # 解包的形式，读取对应的值

    # 相对路径 vs 绝对路径

    • 绝对路径，是文件真正存在的路径
    • 相对路径, 是以当前目录为基准，进行一级一级的这个目录推导的一个路径
    • 我们使用./ 来表示当前所在目录
    • 使用../ 来表示上一级目录

    • 那就是我们给到新名字参数的时候，没有包含路径，直接给了一个 文件名，
    from pathlib import Path

p = Path('D:/ghh/model/blog/model-blog/docs/pystudy')

n = p / "hm.txt"

# n.mkdir(parents=True, exist_ok=True)
# 因为没有指定路径，所以直接跑到了 项目根目录下，而且生成的是文件夹，并不是名字
n.rename("newHm.txt") # 这个写法，不仅没有看到 newHm.txt， 还把 hm.txt 干没了

    # 删除操作

    • rmdir(), 删除文件夹;
    • unlink(), 删除文件
    1. rmdir() 删除文件夹的时候，如果文件夹里有文件(会提示，目录不是空的)
    2. 需要先使用unlink() 函数，将文件给删除
    from pathlib import Path

p = Path('D:/ghh/model/blog/model-blog/docs/pystudy')

n = p / "hm.txt"

print(n.parent)
# n.parent.rmdir() # 切记不要删掉博客的在用的文件夹，可以新建个项目，验证这个点
n.unlink()

    # glob()

    功能强大的查找功能

    • 查找当前文件夹下面的所有以.txt为后缀的文件
    from pathlib import Path

p = Path('.')
# 查找当前文件夹下面的所有以.txt为后缀的文件
print(p.glob("*.md")) # 给到的是一个：<map object at 0x0000016038C16B60>

ps = p.glob("*.md")
# 可以将ps 变成一个列表的形式, list(ps)
# 可以对 ps 进行遍历
    • 查找当前目录下的下一级目录中的所有.py后缀的这个文件
    from pathlib import Path

p = Path('.')

ps = list(p.glob("*/*.py"))
    • 如果希望进行向下的递归搜索, 也就是：查找当前目录以及该目录下面的所有子目录
    • 可以使用两个星号来表示
    from pathlib import Path

ps = list(p.glob("**/*.py"))
