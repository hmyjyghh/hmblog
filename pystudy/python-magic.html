    Python 里的魔法

    寒梦

    # Python 里的魔法

    # 类里面的魔法方法

    1. __init__
    2. __new__
    3. __del__

    # init

    • 语法
    __init__(self[, ...])
    • 能够在对象初始化的时候，实现个性化定制
    • 它的超能力就是：在类实例化对象的时候，自动进行调用(ps: 我们自己写的，自定义函数，可没有这个功能)

    其实，从对象创建到销毁，Python都提供了相对应的魔法方法

    • 通过对魔法方法进行重写, 我们就可以根据需求，自行修改
    • 学会面向对象编程，可以实现很多功能
    • 学会魔法方法则是让你站在人生巅峰，因为对象听你的

    # new

    • __init__()虽然说是对象构建的时候调用的方法, 但参与构建对象的还有一个__new__()方法

    • 它是在__init__()之前被调用的, 事实上对象是由__new__()方法来创建的

    • 语法, cls 是类

    __new__(cls[, ...])
    • 所以, 我们看到它的第一个参数是cls, 是类，而不是self
    • 所以对象的诞生流程是先调用__new__()方法，创建一个类的实例
    • 然后将其传递给__init__()方法，再进行个性化定制这么一个流程
    • 当我们用类，实例化一个对象的时候，事实上这个对象是由__new__ 方法，创建的

    那么，是哪一个魔法方法创建了实例对象呢? 或者说，实例化对象中调用的第一个魔法方法是什么?

    • 答案必须是: __new__() 方法, 因为__init__() 方法中的self 就是new 返回的，有了self才有这个对象
    • 才有了这个方法跟对象的绑定
    • 但是，需要重写__new__() 方法的情况其实极少
    • 有2种情况
    1. 一种情况是在元类中去定制类
      • 元类, 是一个更高级的概念了, 是Python中最难理解的概念之一, 被称之为魔法方法背后的魔法方法
    2. 另一种情况是在继承不可变数据类型的时候, 如果我们想做一些改动，就可以通过重写__new__()方法进行拦截
    class CapStr(str):
    def __new__(cls, string):
        string = string.upper()
        return super().__new__(cls, string)
    
cs = CapStr("Hmyjy")

print(cs) # HMYJY
    • 无论我们参数中传入的是大写字母还是小写字母，拿到手的都是一个由大写字母组成的字符串对象
    • 这里，之所以能够对不可变对象进行修改，是因为我们赶在实例对象被创建之前就进行了拦截
    • 然后，我们才去调用这个super().__new__()去创建真正的实例
    • 由于CapStr类是继承自str字符串类的, 所以字符串该有的方法它也一并继承了下来
    class CapStr(str):
    def __new__(cls, string):
        string = string.upper()
        return super().__new__(cls, string)
    
cs = CapStr("Hmyjy")

# 比如要把cs 变为小写
print(cs.lower()) # hmyjy

# 再比如把cs 变为首字母大写

print(cs.capitalize()) # Hmyjy

    # del

    • 讲完了对象的构造，那么删除的时候呢

    • 当对象即将被销毁的时候, 也会调用一个魔法方法，叫做: __del__(self)

    • 语法

    __del__(self[, ...])
    • Python跟大多数面向对象的编程语言一样, 都是引入了垃圾回收机制garbage collection
    • 简答来说，就是：当检测到一个对象没有任何引用的时候，才会将它销毁
    • 所以我们只要在del c之前将c给送出去(也即是，将c 赋值给其他变量)
    • 那么del c理论上就不会调用到__del__()魔法方法了, 因为，此时这个对象还存在引用
    class C:
    def __init__(self): 
        print("我来了~") 
    def __del__(self): 
        print("我走了~")
c = C()
d = c # 将c的引用给了d

# 但是这一步，竟然打印了`我走了~`, 可能跟：在交互式环境中，当程序执行完毕后，解释器会清理所有对象
del c # 删除c 的话，不会执行__del__()魔法方法, 因为，此时这个对象还存在引用


# 但是del d, 就会执行__del__()魔法方法了

# del d

    # Python的引用计数机制

    class C:
    def __init__(self): 
        print("我来了~") 
    def __del__(self): 
        print("我走了~")

print("开始执行...")
c = C()      # 输出：我来了~
d = c        # 引用计数=2
print("删除c...")
del c        # 引用计数=1，没有输出！
print("c已删除，但对象还在")
print("删除d...")
del d        # 引用计数=0，输出：我走了~
print("程序结束")

    # 正确的理解

    c = C()  # 引用计数=1
d = c    # 引用计数=2

del c    # 引用计数=1 → 对象还在，不执行__del__
# 此时可以通过 d 访问对象

del d    # 引用计数=0 → 执行__del__

    # 烧脑环节，对象的重生

    • del方法可以通过创建一个该实例的新引用来推迟其销毁, 这也被称之为对象的重生(引用自官方文档)
    • 虽然对象通过del语句删除了，但如果能够在对象被销毁之前，将self给送出去，那么这个对象就算是得以重生了
    • 销毁一个对象，是del加上一个对象名
    • del x 是一个语句，我们没办法执行这个语句的同时，又把它的返回值赋值给另一个变量去存储
    • 那么通过返回值把这个self给送出去，这条路就行不通了

    方法其实也是一个函数, 只是多了一个对象绑定的操作和参数

    • 所以可以使用： global语句
    class D:  
    def __init__(self, name):  
        self.name = name  
    def __del__(self):  
        global x
        x = self

d = D("寒梦又见远")
# print(d.name)
del d # 就是把这个对象d给删除了

# print(d) # NameError: name 'd' is not defined. Did you mean: 'D'?

# 但是由于我们在__del__()这个魔法方法里面, 创建了一个全局变量x
# 我们可以访问到x
print(x) # <__main__.D object at 0x000001CC6AC170E0>, 它事实上就是这个对象d

# 因为它在销毁的时候把self又给送出来了

print(x.name) # 寒梦又见远

    # 还有一种方法

    • 非必要，尽量不要使用全局变量，因为会污染命名空间
    • 另外一种方式实现，我们是不是可以在对象被销毁之前，让它去调用一个函数
    • 然后通过参数传递的形式将self给传出去

    因此，另外一种方法就是通过函数调用的形式, 将这个对象，将这个self给保存起来

    # 在__init__()方法里，实现
class E:
    def __init__(self, name, func):
        self.name = name
        self.func = func
    def __del__(self):
        self.func(self) # 我们可以通过self.func()去调用这个函数， 同时把这个self通过参数的形式给传递进去
    • 我们可以通过self.func()去调用这个函数
    • 同时把这个self通过参数的形式给传递进去

    # 回想下闭包

    • Python闭包会通过某种方式， 将外层函数的局部变量给保存下来
    • 我们这里来定义一个闭包函数
    class E:
    def __init__(self, name, func):
        self.name = name
        self.func = func
    def __del__(self):
        self.func(self)

def outter():
    x = 0 # 外部函数的局部变量
    def inner(y=None):
        nonlocal x # 声明要使用外部函数的 x
        if y:
            x = y # 修改外部函数的 x
        else:
            return x
    return inner


f = outter()
e = E("hmyjy", f)

print(e.name)

del e

g = f()
print(g) # <__main__.E object at 0x000001FB82E67230>, 跟删除之前的e 是一样的
# 也就是 e 初始化的name = 'hmyjy' 的对象又重生了
    • 这里定义为闭包函数的原因，是为了让self保存在外部函数的x变量中
    • 内部函数的作用呢，是用于窃取这个self对象
    • 在__del__()魔法方法中, 调用它的时候是带参数的, 它就把这个参数给存储起来
    • 如果在外部调用这个函数, 它不带参数，就像上面示例中的g = f(), 它不带参数，那就是使用默认值None
    • 那么它就会返回刚刚拿到的self对象

    # 运算相关的魔法方法大合集

    1. Python的除法其实还细分为真除法和地板除两种
    • 真除法，/
    • 地板除, //
    1. 还有求余数、幂运算 等

    重写__add__

    class S(str):
    def __add__(self, other):
        return len(self) + len(other) # self 是自己(+左边的对象)， other 是别人(也就是加号右边的对象)

s1 = S("Hm")
s2 = S("yjy.")

print(s1 + s2) # 6

# 这个__add__重写后，是 + 前面这个对象的方法，而不是+后面的
# 根据示例，可看出来
print(s1 + "Python") # 8
print("Python" + s1) # PythonHm
    • 重写之后，不要拼接的效果了，而是计算2者的长度相加之后的值
    • 也就是两个字符串的总字符个数

    魔法方法的作用，就是拦截。

    重写__radd__, 它是反算术运算的魔法方法

    • __radd__(self, other), x+y(由运算符右侧的对象y触发)
    __radd__方法调用前提: 
当两个对象相加的时候, 如果左侧的对象和右侧的对象不同类型, 
并且左侧的对象没有定义__add__()方法, 或者其__add__()返回NotImplemented, 
那么Python就会去右侧的对象中找查找是否有_radd_()方法的定义。

2个条件
1. + 左侧、右侧 是不同对象
2. 且 + 左侧的对象，没有定义__add()__方法 or  __add__()返回NotImplemented

    class S1(str):
    def __add__(self, other):
        return NotImplemented # 代表是未实现的方法, 明确代表未实现

class S2(str):
    def __radd__(self, other):
        return len(self) + len(other)

s1 = S1("Apple")
s2 = S2("Banana")
print(s1 + s2) # 11

"""
s1 重写了 __add__
s2 重写了 __radd__

1. 还是得到两个字符串的一个总字符个数
2. 这里能够成功地调用到S2的_radd__()方法，首先， 是因为S2实现了_radd__()方法, 其次 S1和S2两个，是基于不同类的对象
3. 且S1里边，必须不能实现__add__() 方法
4. 不然，还是会优先去执行左侧对象的__add__() 方法
5. 这里, 我们的__add__() 方法返回这个 Notlmplemented, 含义: 就是明确表示这个方法未实现
6. 或者，我们S1不写这个__add__() 方法，直接来个pass, 输出的也是两者字符串字符数的长度。
"""
    • i 开头的，也就是增强赋值运算符, 这个执行的是运算兼赋值的操作
    • +=, 也就是说它会修改对象自身, 重新定一个同名的类就把原来的给覆盖了
    class S1(str):
    def __iadd__(self, other):
        return len(self) + len(other)

class S2(str):
    def __radd__(self, other):
        return len(self) + len(other)

s1 = S1("Apple")
s2 = S2("Banana")

# print() 函数只能接收表达式作为参数
# 赋值语句（如 s1 += s2）是一个语句，不是表达式，不能作为函数参数
# print(s1 += s2)

# 方法1：先执行赋值，再打印
s1 += s2 # 相当于: s1 = s1.__iadd__(s2)
print(s1) # 11
print(type(s1)) # <class 'int'>, 变成了一个 int  类型的
    • 使用增强赋值运算符一定要先想清楚，别把自己的对象给搞丢了，也就是类型被改变了
    • +=的左侧的这个对象没有实现__iadd__()方法, 那么Python 就会退而求其次，使用相应的_add_0方法和_radd_0方法来替代
    • s2 没有实现add 却有radd 方法
    class S2(str):
    def __radd__(self, other):
        return len(self) + len(other)

s2 = S2("Banana")
s2 += s2
print(s2) # 现在s2就变成了两个'Banana'加到一块了是不是
# 再来看下s2的类型，type(s2)
# 但是不是应该计算两者的字符数吗
# 毕竟我们在S2里边它有去定义这个__radd__()
# 它的__radd__(), 就是：len(self) + len(other)
# 因为要成功的调用__radd__()这个方法

"""
它是有条件的，
1. 首先, 两个对象要基于不同的类
2. 这里S2和S2都是相同的类，所以它不会去调用这个__radd__()方法，
3. 而是去它的父类，它的str 这个父类里面去查找_add__（）方法
4. 得到的就是字符串的拼接， 
"""
    • 魔法方法并不仅仅只能拦截运算符， 它连发生在对象的内置函数， 比如说这个__int__函数，
    • 通常, 如果我们希望将一个对象转换为整数的形式，那么就会想到使用这个__int__()这个内置函数，它事实上也有对应的拦截的魔法方法，我们就可以轻松的去实现一个将中文转换为整数的对象
    • 默认的int()函数就能够实现，这里我们是对__int__()进行了拦截，在它原来的基础上增加了对中文的识别
    class ZH_INT:
    def __init__(self, num):
        self.num = num

    def __int__(self):
        try:
            return int(self.num)
        except ValueError:
            zh = {
                "零": 0, "一": 1, "二": 2, "三": 3, "四": 4, "五": 5, "六": 6, "七": 7, "八": 8, "九": 9,
                "壹": 1, "贰": 2, "叁": 3, "肆": 4, "伍": 5, "陆": 6, "柒": 7, "捌": 8, "玖": 9
            }
            result = 0

            for each in self.num:
                if each in zh:
                    result += zh[each]
                else:
                    result += int(each)
                print(result) # 1、10、100
                result *= 10
            return result // 10

# n = ZH_INT("520")
# print(int(n)) # 520, 默认的int函数就能实现

n1 = ZH_INT("壹零零")
print(int(n1)) # 100
n2 = ZH_INT("五贰零")
print(int(n2)) # 520

class S2(str):
