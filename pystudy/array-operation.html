    按照固定字符数进行切分

    寒梦

    # 按照固定字符数进行切分

    import re

def split_by_fixed_char(string, char_num):
  return [string[i:i+char_num] for i in range(0, len(string), char_num)]

# 测试
string = "1234567890"
char_num = 3
result = split_by_fixed_char(string, char_num)
print(result)  # 输出: ['123', '456', '789', '0']

    # 按照固定字符数进行切分，不足部分用0填充

    

在Python中，`range(0, len(string), char_num)` 是一个`range()`函数调用，用于生成一个数字序列。这个序列通常用于`for`循环中控制迭代。

## 参数含义

`range(start, stop, step)` 有三个参数：

1. **`start`** (第一个参数 `0`) - **起始值**
  - 序列开始的数字（包含）
  - 默认是 `0`，可以省略

2. **`stop`** (第二个参数 `len(string)`) - **结束值**
  - 序列结束的数字（不包含）
  - 这是唯一必须提供的参数

3. **`step`** (第三个参数 `char_num`) - **步长**
  - 每次递增的大小
  - 默认是 `1`，可以省略
  - 可以是正数（递增）或负数（递减）

## 具体示例

```python
string = "HelloWorld"
char_num = 3

# 这会生成序列: 0, 3, 6, 9
for i in range(0, len(string), char_num):
  print(f"索引 {i}: 字符 '{string[i]}'")
  print(f"子字符串: '{string[i:i+char_num]}'")

# 你可以用一个通俗易懂的例子，给我讲一下这2层遍历的作用吗

    用一个点外卖的例子来帮你理解这两层遍历，非常直观：

    # 🍔 场景设定：外卖订单检查系统

    想象你是一个外卖平台的质量检查员，要检查送餐员送的对不对：

    • predictions = 送餐员实际送的餐（预测结果）
    • labels = 顾客真正点的餐（真实标签）
    • -100 = 顾客没点的东西（比如餐厅赠送的小菜，不需要检查）

    # 📦 具体数据示例

    # 预测结果：送餐员送了啥
predictions = [
    [1, 2, 3, 4],    # 第一个订单：汉堡, 薯条, 可乐, 冰淇淋
    [2, 1, 4, 3]     # 第二个订单：薯条, 汉堡, 冰淇淋, 可乐
]

# 真实标签：顾客点了啥（-100表示没点）
labels = [
    [1, 2, 3, -100],  # 第一个顾客：点了汉堡,薯条,可乐，没点冰淇淋
    [2, -100, -100, 3] # 第二个顾客：点了薯条,可乐，没点汉堡和冰淇淋
]

# 菜单映射
label_list = ["", "汉堡", "薯条", "可乐", "冰淇淋"]  # 索引0空着不用

    # 🔍 两层遍历分解

    # 第一层遍历：检查每个订单

    for prediction, label in zip(predictions, labels):
    # 第一次循环：prediction=[1,2,3,4], label=[1,2,3,-100]
    # 第二次循环：prediction=[2,1,4,3], label=[2,-100,-100,3]

    # 第二层遍历：检查订单里的每个商品

    [p for p, l in zip(prediction, label) if l != -100]

    第一个订单的检查过程：

    预测(p) 真实(l) 是否需要检查？ 结果
    1(汉堡) 1(汉堡) ✅ (1 != -100) 保留
    2(薯条) 2(薯条) ✅ (2 != -100) 保留
    3(可乐) 3(可乐) ✅ (3 != -100) 保留
    4(冰淇淋) -100(没点) ❌ (-100) 过滤掉

    → 结果：[1, 2, 3] → 转成文字：["汉堡", "薯条", "可乐"]

    第二个订单的检查过程：

    预测(p) 真实(l) 是否需要检查？ 结果
    2(薯条) 2(薯条) ✅ (2 != -100) 保留
    1(汉堡) -100(没点) ❌ (-100) 过滤掉
    4(冰淇淋) -100(没点) ❌ (-100) 过滤掉
    3(可乐) 3(可乐) ✅ (3 != -100) 保留

    → 结果：[2, 3] → 转成文字：["薯条", "可乐"]

    # 🎯 最终结果

    true_predictions = [
    ["汉堡", "薯条", "可乐"],  # 第一个订单的有效预测
    ["薯条", "可乐"]           # 第二个订单的有效预测
]

    # 💡 核心作用总结

    1. 外层遍历：for prediction, label in zip(predictions, labels)

      • 像质检员逐个订单检查
      • 确保预测和真实标签一一对应

    2. 内层遍历：for p, l in zip(prediction, label) if l != -100

      • 像质检员检查订单里每个商品
      • 只关注顾客真正点了的商品（过滤掉-100）
      • 避免把"餐厅赠送品"也算进准确率

    3. 转换：label_list[p]

      • 把数字编号转成看得懂的文字（1→"汉堡"）

    # 在NER任务中的实际意义

    在命名实体识别中：

    • -100 对应的是 [CLS], [SEP], 子词的后半部分等
    • 这样处理后，只评估真正有意义的实体标签
    • 计算准确率时更公平、更准确

    # 列表推导式

    列表推导式的优势：比普通循环更简洁、执行效率更高；

    high = [d for s, d in zip(scores, docs) if s >= high_threshold]

    这个列表推导式的核心作用就是过滤 + 取值——只有当 s >= high_threshold 成立时，才会把对应的 d 加入最终的列表 high，本质就是对 docs 按照 scores 的阈值进行过滤。

    # usage

    假设：

    scores = [85, 92, 78, 95]  # 分数列表
docs = ["文档1", "文档2", "文档3", "文档4"]  # 对应文档
high_threshold = 90  # 高阈值

    执行列表推导式后：

    • 第1对：s=85 < 90 → 跳过，不加入d（文档1）
    • 第2对：s=92 ≥ 90 → 加入d（文档2）
    • 第3对：s=78 < 90 → 跳过，不加入d（文档3）
    • 第4对：s=95 ≥ 90 → 加入d（文档4）

    最终 high = ["文档2", "文档4"]，完美验证了“过滤后取d”的逻辑。

    扩展：如果想同时保留s和d，可以把d换成(s, d)，最终得到[(92, "文档2"), (95, "文档4")]。
