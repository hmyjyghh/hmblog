    # 函数相关

    # 位置参数

    • 按形参位置, 传入实参

    # 位置参数问题

    def myfunc(s="苹果", vt, o="寒梦"):
    return "".join((o, vt, s))

# vt 的位置，报错信息(会有红色波浪线)：Non-default argument follows default argument
# 非默认参数出现在默认参数之后

    Python 有明确的函数参数定义规则：默认参数（带=赋值的参数，如s="苹果"、o="寒梦"）必须位于非默认参数（不带=赋值的参数，如vt）之后。

    # 两种修改方式

    • 方案 1：调整参数顺序（推荐，不改变函数使用逻辑）
    # 调整后：非默认参数vt放在默认参数s、o之前
def myfunc(vt, s="苹果", o="寒梦"):
    return "".join((o, vt, s))

# 测试使用（正常运行）
result = myfunc("吃")
print(result)  # 输出：寒梦吃苹果
    • 方案 2：给vt也设置默认值（适用于需要保留原有参数顺序的场景）
    # 给vt添加默认值，所有参数均为默认参数，顺序可保留
def myfunc(s="苹果", vt="喜欢", o="寒梦"):
    return "".join((o, vt, s))

# 测试使用（正常运行）
result1 = myfunc()  # 使用所有默认值
result2 = myfunc("香蕉", "吃")  # 自定义s和vt
print(result1)  # 输出：寒梦喜欢苹果
print(result2)  # 输出：寒梦吃香蕉

    # 示例二

    # 闭包

    1. 函数内部可以访问外部函数的变量(涉及到一个作用域的问题)
    2. 返回的是一个函数

    # 核心技巧

    1. 利用嵌套函数的外层作用域具有记忆能力这个特性, 让数据保存在外层函数的参数或者变量中
    2. 将内层函数作为返回值给返回，这样就可以从外部间接的调用到内层的函数

    这2个点就是闭包的生成原理

    # nonlocal

    • nonlocal语句, 如果我们非要在内部函数去修改外部函数的变量
    • 了解即可，轻易不要用
    def funA():
    x = 520
    def funB():
        nonlocal x # 可理解为，不使用全局变量
        x = 880
        print("In funB, x =", x)
    funB()
    print("In funA, x =", x)

funA()
# In funB, x = 880
# In funA, x = 880

    # LEGB 规则

    掌握了LEGB 规则，相当于掌握了：Python 变量的解析机制

    • Local
    • Enclosed, 是嵌套函数的外层函数作用域
    • Global, 全局作用域
    • Build-In， 也叫内置作用域

    # 优先级，G > B

    • 当局部作用域与全局作用域发生冲突的时候, Python会使用局部作用域的变量, 除非你使用Global语句, 进行特别的声明
    • 然后当函数嵌套发生的时候, 局部作用域又会覆盖外层函数的这个作用域, 除非你使用nonlocal语句进行声明

    切记：不要使用内置函数名，作为变量名

    str = '变量名使用了python中的内置函数，引发的问题'

str(520) # TypeError: 'str' object is not callable

# LEGB 法则，G > B， G 就把B 给覆盖了，str 的Build-In， 就被毁了

    1. 写一个，时间管理大师的函数

    通过time模块的time()函数运行时间的一个方式，

    import time

def time_master(func):
    print("开始运行程序..")
    start = time.time()
    func()
    stop = time.time() # 计算的是时间戳
    print("结束程序运行..")
    print(f"一共耗费了 {(stop-start): .2f} 秒。")
    
def myfunc():
    time.sleep(2)
    print("Hello Hmyjy.")

# 主动调用time_master 函数
time_master(myfunc)
# 开始运行程序..
# Hello Hmyjy.
# 结束程序运行..
# 一共耗费了  2.00 秒。

    • 计算耗时思想, 函数执行前,打印时间

    • 函数执行后，再次打印时间（不是接口请求的话，可以加延迟2秒后，再执行函数）

    • 两者的时间差，就是函数执行所消耗的时间，也就是函数运行的时间。

    • 那么我们这个time_master()函数

    • 那么它最后得到的时间呢

    • 但是想要打印函数运行，需要的时间, 都要显示的去调用这个time_master()函数才行

    • 这就是装饰器存在的意义

    • 更好的方案是在调用myfunc()函数的时候，它能够自觉地去执行time_master()函数

    • 到这里，引入了装饰器的概念

    学会了Python的装饰器，很有用，能做很多事情

    • 装饰器有2个特性
    1. 将内层函数作为返回值返回
    2. 拿函数当参数

    2者综合起来，就是装饰器

    • @ + 装饰器的名字，这是一个语法糖
    import time


def time_master(func):
    def call_fun():
        print("开始运行程序..")
        start = time.time()
        func()
        stop = time.time()  # 计算的是时间戳
        print("结束程序运行..")
        print(f"一共耗费了 {(stop-start): .2f} 秒。")
    return call_fun

@time_master
def myfunc():
    time.sleep(2)
    print("Hello Hmyjy.")

myfunc()
# 开始运行程序..
# Hello Hmyjy.
# 结束程序运行..
# 一共耗费了  2.00 秒。

    return inner

    return 2

    # 带参数的装饰器

    • 如何给装饰器传递参数呢?

    多加了一层函数嵌套, 也就是 def logger(msg):

    import time

def logger(msg):

        def call_func():
            start = time.time()
            func()
            stop = time.time()
            print(f"[{msg}]—共耗费了 {(stop-start):.2f}")
        return call_func
    return time_master

# @logger(msg="A")

    # 汉诺塔

    它就是一个典型的不能按照常规套路出牌的游戏

    • 必须先想办法，把压在身上的其他圆盘，给挪开
    • 如此往复，直到遇到最顶上的一个圆盘，才可以真正地去移动它。

    # 游戏规则

    • 规则一: 一次只能移动一枚金片
    • 规则二: 小片必须在大片的上面

    # 思路，先用最简单的3片圆盘，来做移动

    • 先借助C，把上面最小的A，移动到C
    • 再把B，移动到中间的位置
    • 再把最小的A，挪到B
    • 然后把最大的圆盘挪到C
    • 再把最小的圆盘，挪到A
    • B挪到C，再把A挪过去(也就是挪到C，C对应的参数为: z)，完成。

    def hannuota(n, x, y, z): 
    if n == 1:  
        print(x, '-->', z)  # 如果只有 1 层，直接将金片从 x 移动到 z  
    else:  
        hannuota(n-1, x, z, y)  # 将 x 上的 n-1 个金片移动到 y  
        print(x, '-->', z)  # 将最底下的金片从 x 移动到 z  
        hannuota(n-1, y, x, z)  # 将 y 上的 n-1 个金片移动到 z

n = int(input('请输入汉语塔的层数：'))
hannuota(n, 'A', 'B', 'C')

    # 代码解释

    1. 分别将ABC传递到x,y,z三个参数中
    2. if n == 1的话，也就是汉诺塔的层数只有一层的时候
    3. 就直接将×上的金片移动到z上面, 也就是从A到C
    4. if n > 1, 那么就会进入到 递归程序里
    5. 先将n-1个金片从x移动到y(也就是把上面的小的，都先挪开，借助一根柱子)
    6. 再将x上剩下的一个金片移动到z
    7. 最后将n-1个金片从y移动到z

    # 代码里的难点

    • 一下子是x, y, z
    • 一下子又是x, z, y
    • 最后又是y, x, z

    # 需要分清

    1. 一定要分清形参和实参
    2. 变来变去的其实是实参
    3. 也就是刚开始我们传递进去的ABC这三个值，在变来变去
    4. ABC 三个值，通过x,y,z来进行一个传递

    # 代码理解

    • 假设n=3, 那么第一次调用, 就是：hanoi(3,'A','B','C')
    • 按照这个思路去捋

    # 偏函数，partial

    1. 偏函数是指对指定的函数进行二次包装
    2. 通常呢是将现有的函数部分参数预先给绑定
    3. 从而得到一个新的函数，该函数我们就称之为偏函数。相比原函数, 偏函数具有较少的可变参数, 从而降低了函数调用的难度。
    4. 偏函数被调用的时候, 如果提供更多的参数, 它们会被附加到func函数的位置参数args中;
    5. 如果额外的关键字参数, 它们会扩展并重载func函数的关键字参数keywords.
    6. 实现原理大致等价于:
    def partial(func, /, *args, **keywords):
    def newfunc(*fargs, **fkeywords):
        newkeywords = {**keywords, **fkeywords}
        return func(*args, *fargs, **newkeywords)
    newfunc.func = func
    newfunc.args = args
    newfunc.keywords = keywords
    return newfunc
