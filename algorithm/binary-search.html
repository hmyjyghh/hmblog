    # 二分查找

    二分查找是一种算法，其输入是一个有序的元素列表（必须有序）。如果要查找的元素包含在列表中，二分查找返回其位置；否则返回null。

    # 1. 题目1

    给定一个 n 个元素有序的（升序）整型数组 nums 和一个目标值 target  ，写一个函数搜索 nums 中的 target，如果目标值存在返回下标，否则返回 -1。

    # 示例 1:

    输入: nums = [-1,0,3,5,9,12], target = 9 输出: 4 解释: 9 出现在 nums 中并且下标为 4

    # 示例 2:

    输入: nums = [-1,0,3,5,9,12], target = 2 输出: -1 解释: 2 不存在 nums 中因此返回 -1

    # 分析如下:

    在升序数组 nums 中寻找目标值 target，对于特定下标 i，比较 nums[i] 和 target 的大小：

    • 如果 nums[i] = target，则下标 i 即为要寻找的下标；

    • 如果 nums[i] > target，则 target 只可能在下标 i 的左侧；

    • 如果 nums[i] < target，则 target 只可能在下标 i 的右侧。

    # 代码示例:

    var search = function(nums, target) {
	let low = 0, high = nums.length - 1;
	while (low <= high) {
		const mid = Math.floor((high - low) / 2) + low; // (high - low) 要查找的索引范围  + low  从上次查找的位置开始
		const num = nums[mid];
		if (num === target) {
			return mid;
		} else if (num > target) {
			high = mid - 1;
		} else {
			low = mid + 1;
		}
	}
	return -1;
};

    # 复杂度分析

    • 时间复杂度：O(logn)，其中 n 是数组的长度。

    • 空间复杂度：O(1)。

    # 2. 题目2

    二分查找版 快速排序

    每次都找一个基准值

    var quickSort = function(arr) {
  if (arr.length <= 1) return arr; //边界条件 
  
  var pivotIndex = Math.floor(arr.length / 2);
  var pivot = arr.splice(pivotIndex, 1)[0]; // 找到基准值
  var left = [];
  var right = [];
  for (var i = 0; i < arr.length; i++) {
    if (arr[i] < pivot) {
      left.push(arr[i]);
    } else {
      right.push(arr[i]);
    }
  }
  return quickSort(left).concat([pivot], quickSort(right)); //递归
};

console.log(quickSort([1,23,5,7,8,44,33,22,3,0]))

    # 2. 题目3

    虚拟列表 非等高 二分查找算法(旧电脑代码)

    1. 问题背景

    虚拟列表通过仅渲染可视区域内的元素来优化性能。对于非等高列表，传统方法（如固定高度计算）无法直接使用，需结合 前缀和数组与二分查找实现高效定位。

    1. 核心思路

    前缀和数组

    • 定义：prefixSum[i] 表示前 i 个元素的总高度。
    • 构建：遍历元素高度数组，累加生成前缀和数组。
    const heights = [100, 200, 150]; // 各元素高度
const prefixSum = [0]; // 前缀和数组，初始为0
for (const h of heights) {
  prefixSum.push(prefixSum[prefixSum.length - 1] + h);
}
// 结果：prefixSum = [0, 100, 300, 450]

    二分查找定位索引

    • 目标：给定一个累积高度 offset，找到最大的 i 使得 prefixSum[i] <= offset。
    • 实现：使用二分查找在 prefixSum 中快速定位索引。
    function findIndexByOffset(offset, prefixSum) {
  let left = 0, right = prefixSum.length - 1;
  while (left <= right) {
    const mid = Math.floor((left + right) / 2);
    if (prefixSum[mid] <= offset) {
      left = mid + 1;
    } else {
      right = mid - 1;
    }
  }
  return right; // 返回最大的i满足prefixSum[i] <= offset
}
    1. 应用场景
    • 场景 1：计算滚动位置对应的起始索引
      • 需求：根据当前滚动偏移量 scrollTop，确定可视区域的起始元素索引。
    const scrollTop = 180; // 当前滚动位置
const startIndex = findIndexByOffset(scrollTop, prefixSum);
// 结果：startIndex = 1（对应prefixSum[1]=100 <= 180 < prefixSum[2]=300）
    • 场景 2：计算元素在滚动容器中的位置
      • 需求：已知元素索引 index，计算其在滚动容器中的绝对位置。
    const index = 2;
const position = prefixSum[index]; // 前两个元素总高度为300
    1. 完整代码
    // 生成前缀和数组
function buildPrefixSum(heights) {
  const prefixSum = [0];
  for (const h of heights) {
    prefixSum.push(prefixSum[prefixSum.length - 1] + h);
  }
  return prefixSum;
}

// 二分查找定位索引
function findIndexByOffset(offset, prefixSum) {
  let left = 0, right = prefixSum.length - 1;
  while (left <= right) {
    const mid = Math.floor((left + right) / 2);
    if (prefixSum[mid] <= offset) {
      left = mid + 1;
    } else {
      right = mid - 1;
    }
  }
  return right;
}

// 示例数据
const heights = [100, 200, 150, 300, 250]; // 非等高元素高度
const prefixSum = buildPrefixSum(heights);

// 测试1：查找scrollTop=400对应的起始索引
const scrollTop = 400;
const startIndex = findIndexByOffset(scrollTop, prefixSum);
console.log(startIndex); // 输出2（前2个元素总高度300 <= 400 < 450）

// 测试2：计算索引3的位置
const index = 3;
const position = prefixSum[index];
console.log(position); // 输出450（前3个元素总高度100+200+150=450）
    1. 优化与注意事项
    2. 数据更新：当元素高度变化时，需重新生成 prefixSum 数组。
    3. 边界处理：
    • 若 offset >= prefixSum[prefixSum.length - 1]，返回最后一个元素索引。
    • 若 offset < 0，返回 -1 或 0（根据业务逻辑调整）。
    1. 性能优化：
    • 对于静态数据，prefixSum 只需生成一次。
    • 对于动态数据，可缓存 prefixSum，仅在数据变化时重新计算。
    1. 扩展应用
    • 动态加载数据：结合分页或无限滚动时，可预计算部分 prefixSum，减少内存占用。
    • 复杂布局：支持元素高度动态变化（如文本撑开）时，需重新计算 prefixSum 并触发重渲染。

    通过前缀和数组与二分查找的结合，非等高虚拟列表的定位和计算效率可达到 O(log n)，显著优于遍历法的 O(n)，适用于大数据量场景。

    # 虚拟列表（非等高）实现详解

    虚拟列表是一种优化长列表渲染性能的技术，特别适用于非等高的列表项场景。下面我将详细介绍如何实现一个非等高的虚拟列表。

    # 核心原理

    非等高虚拟列表的核心原理是：

    1. 只渲染可视区域内的列表项，避免渲染所有项导致的性能问题
    2. 动态计算每个列表项的位置和高度，实现精确滚动定位
    3. 使用占位元素维持滚动条的正确性，保持原生滚动体验

    # 实现步骤

    # 1. 数据结构设计

    首先需要存储每个列表项的高度信息：

    // 列表项数据
const items = [
  { id: 1, content: 'Item 1' },
  { id: 2, content: 'Item 2' },
  // ...
];

// 存储每个项的高度和位置
const itemPositions = items.map((item, index) => ({
  id: item.id,
  // 初始可以设置一个预估高度，实际渲染后更新
  height: 80, // 默认高度
  top: index * 80, // 初始top位置
  bottom: (index + 1) * 80 // 初始bottom位置
}));

    # 2. 计算可视区域

    // 容器高度
const containerHeight = 500;
// 滚动位置
let scrollTop = 0;

// 计算可视区域的起始和结束索引
const getVisibleRange = () => {
  const startIndex = findNearestItemIndex(scrollTop);
  const endIndex = findNearestItemIndex(scrollTop + containerHeight);
  return { startIndex, endIndex };
};

// 二分查找最近的项
const findNearestItemIndex = (position) => {
  let low = 0;
  let high = itemPositions.length - 1;
  
  while (low <= high) {
    const mid = Math.floor((low + high) / 2);
    const { top, bottom } = itemPositions[mid];
    
    if (position < top) {
      high = mid - 1;
    } else if (position > bottom) {
      low = mid + 1;
    } else {
      return mid;
    }
  }
  
  return Math.max(0, Math.min(itemPositions.length - 1, low));
};

    # 3. 动态测量高度

    // 渲染后更新实际高度
const updateItemSize = (id, height) => {
  const index = itemPositions.findIndex(item => item.id === id);
  if (index === -1 || itemPositions[index].height === height) return;
  
  // 更新当前项高度
  const diff = height - itemPositions[index].height;
  itemPositions[index].height = height;
  itemPositions[index].bottom = itemPositions[index].top + height;
  
  // 更新后续项的位置
  for (let i = index + 1; i < itemPositions.length; i++) {
    itemPositions[i].top = itemPositions[i - 1].bottom;
    itemPositions[i].bottom = itemPositions[i].top + itemPositions[i].height;
  }
};

    # 4. 渲染实现（React示例）

    function VirtualList({ items, containerHeight }) {
  const [scrollTop, setScrollTop] = useState(0);
  const [itemPositions, setItemPositions] = useState(() => 
    items.map((item, index) => ({
      id: item.id,
      height: 80, // 预估高度
      top: index * 80,
      bottom: (index + 1) * 80
    }))
  );
  
  // 获取可视区域项
  const { startIndex, endIndex } = getVisibleRange(itemPositions, scrollTop, containerHeight);
  const visibleItems = items.slice(startIndex, endIndex + 1);
  
  // 总高度（用于容器占位）
  const totalHeight = itemPositions[itemPositions.length - 1]?.bottom || 0;
  
  // 更新项高度
  const handleItemResize = (id, height) => {
    setItemPositions(prev => {
      const newPositions = [...prev];
      const index = newPositions.findIndex(item => item.id === id);
      
      if (index === -1 || newPositions[index].height === height) return prev;
      
      // 更新当前项
      newPositions[index].height = height;
      newPositions[index].bottom = newPositions[index].top + height;
      
      // 更新后续项
      for (let i = index + 1; i < newPositions.length; i++) {
        newPositions[i].top = newPositions[i - 1].bottom;
        newPositions[i].bottom = newPositions[i].top + newPositions[i].height;
      }
      
      return newPositions;
    });
  };
  
  return (
    <div 
      style={{ height: containerHeight, overflow: 'auto' }}
      onScroll={(e) => setScrollTop(e.currentTarget.scrollTop)}
    >
      <div style={{ height: totalHeight, position: 'relative' }}>
        {visibleItems.map(item => {
          const position = itemPositions.find(pos => pos.id === item.id);
          return (
            <div
              key={item.id}
              style={{
                position: 'absolute',
                top: `${position.top}px`,
                width: '100%'
              }}
            >
              <ListItem 
                content={item.content}
                onResize={(height) => handleItemResize(item.id, height)}
              />
            </div>
          );
        })}
      </div>
    </div>
  );
}

// 列表项组件（带高度测量）
function ListItem({ content, onResize }) {
  const ref = useRef(null);
  
  useEffect(() => {
    if (ref.current) {
      const height = ref.current.getBoundingClientRect().height;
      onResize(height);
    }
  }, [content, onResize]);
  
  return (
    <div ref={ref} style={{ padding: '16px', borderBottom: '1px solid #eee' }}>
      {content}
    </div>
  );
}

    # 性能优化技巧

    1. 预渲染额外项：可视区域上下多渲染几项，避免快速滚动时空白

      const buffer = 5; // 缓冲项数
const startIndex = Math.max(0, findNearestItemIndex(scrollTop) - buffer);
const endIndex = Math.min(
  itemPositions.length - 1, 
  findNearestItemIndex(scrollTop + containerHeight) + buffer
);

    2. 节流滚动事件：避免频繁更新

      const handleScroll = throttle((e) => {
  setScrollTop(e.currentTarget.scrollTop);
}, 16); // 约60fps

    3. 缓存计算结果：使用 useMemo 缓存可见项计算

      const visibleItems = useMemo(() => {
  return items.slice(startIndex, endIndex + 1);
}, [items, startIndex, endIndex]);

    4. 预估高度优化：根据内容类型设置更准确的预估高度，减少布局抖动

    # 现有库推荐

    如果不想自己实现，可以考虑以下成熟库：

    • react-window（FixedSizeList 的变体）
    • react-virtualized（功能更全面）

    # 总结

    非等高虚拟列表实现的关键点：

    1. 维护每个项的精确位置信息
    2. 动态测量和更新项的高度
    3. 高效计算可视区域
    4. 正确处理滚动和渲染

    这种实现方式可以显著提升长列表的性能，特别是在移动端或数据量大的场景下。

    function binarySearch(list, value) {
  let start = 0;
  let end = list.length - 1;
  let tempIndex = null;
  while (start <= end) {
    let midIndex = parseInt((start + end) / 2);
    let midValue = list[midIndex].bottom;
    if (midValue === value) {
      return midIndex + 1;
    } else if (midValue < value) {
      start = midIndex + 1;
    } else if (midValue > value) {
      if (tempIndex === null || tempIndex > midIndex) {
        tempIndex = midIndex;
      }
      end = end - 1;
    }
  }
  return tempIndex;
}
