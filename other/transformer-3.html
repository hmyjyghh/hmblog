    # 设置随机种子，保证结果可复现
def set_seed(seed=42):
  torch.manual_seed(seed)
  torch.cuda.manual_seed_all(seed)
  np.random.seed(seed)
  torch.backends.cudnn.deterministic = True

    设置随机种子在大模型训练中极其重要，它的主要作用是保证实验的可复现性（Reproducibility）。

    # 为什么需要设置随机种子？

    深度学习训练过程中涉及大量的随机性，包括：

    1. 权重初始化：神经网络参数的初始值是随机生成的
    2. 数据 shuffling：数据加载时的随机打乱顺序
    3. Dropout：随机丢弃神经元
    4. 数据增强：随机变换（如随机裁剪、旋转等）
    5. 优化器：某些优化算法中的随机因素
    6. CUDA 操作：GPU 计算中的并行操作可能引入不确定性

    如果没有设置随机种子，每次运行程序都会得到不同的随机数序列，导致：

    • 每次训练的损失曲线不同
    • 最终模型性能有差异
    • 无法准确比较不同超参数或模型架构的效果

    # 你提供的代码详解

    def set_seed(seed=42):
  torch.manual_seed(seed)          # 设置 PyTorch CPU 的随机种子
  torch.cuda.manual_seed_all(seed) # 设置所有 GPU 的随机种子
  np.random.seed(seed)             # 设置 NumPy 的随机种子
  torch.backends.cudnn.deterministic = True  # 确保 CUDA 卷积操作 deterministic

    # 每行代码的作用：

    1. torch.manual_seed(seed)：
    • 设置 PyTorch 在 CPU 上的随机数生成器种子
    • 影响所有在 CPU 上进行的随机操作
    1. torch.cuda.manual_seed_all(seed)：
    • 设置所有 GPU 的随机种子
    • 确保在多 GPU 训练时，所有 GPU 的随机行为一致
    1. np.random.seed(seed)：
    • 设置 NumPy 的随机种子
    • 影响任何使用 NumPy 的随机操作（如数据预处理）
    1. torch.backends.cudnn.deterministic = True：
    • 这是最关键的一步！
    • CUDA 的 cuDNN 库为了性能优化，会使用非确定性的算法
    • 设置为 True 会强制使用确定性算法，牺牲一些性能来保证可复现性

    # 使用方法

    # 在训练开始前调用
set_seed(42)  # 42是常用的默认值，但可以是任何整数

# 然后进行模型初始化、数据加载、训练等操作
model = YourModel()
train_loader = DataLoader(...)

# 训练循环...

    # 注意事项

    1. 性能代价：torch.backends.cudnn.deterministic = True 可能会使训练速度变慢10-30%，因为禁用了某些优化。

    2. 不完全确定性：即使在设置了所有这些之后，由于硬件和软件栈的复杂性，100% 的完全复现仍然很难保证，特别是在多GPU训练中。

    3. 随机种子的选择：种子值本身不重要（42是惯例），重要的是每次实验使用相同的种子。

    4. 实验对比：当比较不同模型或超参数时，应该在相同的随机种子下进行，这样才能公平比较。

    # 实际应用场景

    # 比较不同学习率的效果
for lr in [1e-3, 1e-4, 1e-5]:
  set_seed(42)  # 每次都用相同的种子
  model = create_model()
  optimizer = Adam(model.parameters(), lr=lr)
  # 训练和评估...
  # 这样得到的性能差异才能真正反映学习率的影响，而不是随机性

    总结：设置随机种子是深度学习研究中保证实验结果可靠、可比较的基础性工作，对于大模型训练尤其重要，因为训练成本极高，必须确保实验结果的可靠性。
