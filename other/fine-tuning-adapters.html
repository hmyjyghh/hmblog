    参数高效微调的核心思想就是不更新整个大模型的参数，只更新一小部分新增的、轻量级的参数。Adapter（适配器）正是实现这一思想的经典方法。

    下面我为你详细讲解如何添加 Adapters，包括其核心思想、具体做法以及代码示例。

    # 一、核心思想：在模型中插入“小模块”

    Adapter 的基本思路非常简单直观：

    1. 冻结主干网络：保持预训练大模型（如 BERT、GPT）的所有原始参数不变（冻结），不进行梯度更新。
    2. 插入适配器模块：在模型的某些层（通常是 Transformer 层的每个前馈神经网络之后）插入一些小型、全连接神经网络。这些就是 Adapters。
    3. 只训练适配器：在微调过程中，只更新这些新插入的 Adapter 模块的参数，而原始模型参数保持固定。

    这样一来，你需要保存和训练的参数量就从整个模型的数十亿个，减少到了只有 Adapter 的几百万甚至几十万个，极大地节省了内存和计算资源。

    # 二、Adapter 的具体结构与做法

    一个标准的 Adapter 模块通常是一个 “瓶颈结构” 的前馈神经网络。

    # 1. 结构分解

    假设我们将 Adapter 插入到 Transformer 层的前馈网络之后。

    • 原始流程：输入 -> Transformer Layer (包括Attention和FFN) -> 输出
    • 加入Adapter后流程：
      1. 输入 -> Transformer Layer -> 中间输出
      2. 中间输出 -> Adapter -> Adapter输出
      3. 最终输出 = 中间输出 + Adapter输出 （这里有一个残差连接）

    Adapter 内部的“瓶颈结构”如下图所示：

    这个结构包含以下步骤：

    1. 向下投影：将一个高维特征（例如 d_model=768）通过一个全连接层投影到一个低维空间（例如 adapter_size=64）。这是一个降维过程，大幅减少了参数量。
      • 公式：h_down = W_down * x （W_down 是 [d_model, adapter_size] 的矩阵）
    2. 非线性激活：使用一个激活函数（如 ReLU 或 GELU）引入非线性。
      • 公式：h_nonlinear = GELU(h_down)
    3. 向上投影：将低维特征再投影回原始的高维空间。
      • 公式：h_up = W_up * h_nonlinear （W_up 是 [adapter_size, d_model] 的矩阵）
    4. 残差连接：将 Adapter 的输出与原始输入相加。这确保了即使 Adapter 被随机初始化，也不会干扰模型原有的知识，保证了训练的稳定性。
      • 公式：output = x + h_up

    # 2. 插入位置

    通常，Adapter 被插入到 Transformer 层的两个核心子层之后：

    • 前馈网络之后：这是最经典和常见的位置。
    • 自注意力机制之后：有时也会在这里插入，或者在两个位置都插入。

    # 三、代码实现示例

    现在，我们来看如何用代码实现。我们将使用非常流行的 transformers 库和 adapter-transformers 库（现在已集成到主库中）。

    # 方法一：使用 peft 库（推荐，现代且统一）

    peft 库是 Hugging Face 官方推荐的参数高效微调库，它统一了 Adapter、LoRA 等多种方法。

    from transformers import AutoModelForSequenceClassification, TrainingArguments, Trainer
from peft import AdapterConfig, get_peft_model
import torch

# 1. 加载预训练模型
model = AutoModelForSequenceClassification.from_pretrained(
    "bert-base-uncased",
    num_labels=2
)

# 2. 定义 Adapter 配置
adapter_config = AdapterConfig(
    mh_adapter=True,    # 在多头注意力后添加Adapter
    output_adapter=True, # 在前馈网络后添加Adapter
    reduction_factor=16, # “瓶颈”缩小因子。如果原始维度是768，则adapter隐藏层为768/16=48
    non_linearity="gelu" # 非线性激活函数
)

# 3. 将模型转换为 PEFT 模型，并添加Adapters
model = get_peft_model(model, adapter_config)

# 4. 打印可训练参数，你会发现它只占模型总参数量的很小一部分
model.print_trainable_parameters()
# 输出示例：trainable params: 894,976 || all params: 109,514,762 || trainable%: 0.8172%

# 5. 正常定义训练参数和训练器
training_args = TrainingArguments(
    output_dir="./output",
    learning_rate=3e-4, # Adapter学习率通常可以设高一点
    per_device_train_batch_size=16,
    num_train_epochs=3,
)

# ... 假设你已经加载了数据集 ‘dataset’

    eval_dataset=dataset["test"],

# 6. 开始训练！只有Adapter参数会被更新
trainer.train()

    # 方法二：使用 adapter-transformers 库（传统方法）

    这是 Adapter 方法的原生实现库，提供了更精细的控制。

    from transformers import AutoModel, AdapterConfig, AdapterType

# 1. 加载模型
model = AutoModel.from_pretrained("bert-base-uncased")

# 2. 动态添加Adapter配置
adapter_config = AdapterConfig.load("pfeiffer", reduction_factor=16) # "pfeiffer"是一种经典配置

# 3. 为模型添加一个名为"my_adapter"的Adapter
model.add_adapter("my_adapter", config=adapter_config)

# 4. 激活该Adapter用于训练
model.train_adapter("my_adapter")

# 5. 现在，在训练过程中，只有"my_adapter"的参数会被更新
# ... (后续的TrainingArguments和Trainer定义与上面类似)

# 6. 保存Adapter
model.save_adapter("./saved_adapter", "my_adapter")

# 7. 加载时
model.load_adapter("./saved_adapter")

    # 四、Adapters 的优势与劣势

    # 优势：

    1. 参数高效：极大地减少了可训练参数量，降低了显存需求。
    2. 模块化：可以为不同任务训练不同的 Adapter，然后像插拔U盘一样在同一个基础模型上切换，实现多任务学习。
    3. 知识保留：由于主干网络被冻结，模型不容易发生“灾难性遗忘”，保留了原有的强大语言能力。
    4. 训练稳定：残差连接保证了训练的稳定性。

    # 劣势：

    1. 推理速度略有下降：虽然参数少了，但模型的前向传播路径增加了（多了Adapter的计算），会导致推理速度比原始模型稍慢一些。
    2. 需要设计结构：需要决定插入位置、瓶颈维度等超参数。

    # 总结

    添加 Adapters 进行参数高效微调，其本质是：在预训练模型的特定层中，插入一个具有“瓶颈结构”的小型神经网络，并通过残差连接将其与主干网络集成。在微调时，只训练这些新增的小型网络，而冻结庞大的原始模型。

    这种方法让你可以在有限的硬件资源（如一张 Colab 的 T4 GPU）上，高效地微调一个巨大的模型，是当前大模型领域不可或缺的关键技术。
