    Vue3 Pinia 和 Vuex

    寒梦

    # Vue3 Pinia 和 Vuex

    Pinia 是 Vue 3 的轻量级状态管理方案，相比 Vuex 有三大优势：

    1. 更简洁的 API：直接修改状态、无需 mutations(直接通过 action 修改)，天然支持 TypeScript；
    2. Pinia：天然支持模块化，每个 store 都是独立的，无需额外配置。
    3. 更小的体积：1KB 左右，性能更优。

    • Pinia：代码体积小，性能高，去除了 Vuex 中一些不必要的概念，如 mutations，使状态更新更直接。

    • Vuex：代码体积相对较大，尤其是在复杂项目中，由于 mutations 的存在，状态更新流程稍显繁琐。

    • Vuex：需通过 modules 选项来实现模块化，配置相对繁琐。

    • Pinia：只有 state、actions、getters

    • 而 Vuex 适合需要严格状态变更流程的项目（如大型遗留系统）。

    • 如果是新项目，可以优先选择 Pinia。

    # 2. 核心概念差异

    # (1) Store 定义方式

    • Pinia：
      使用 defineStore 定义 Store，类似 Composition API 的 setup。

      // stores/counter.js
import { defineStore } from 'pinia';

export const useCounterStore = defineStore('counter', {
  state: () => ({ count: 0 }),
  actions: {
    increment() { this.count++; }
  },
  getters: {
    doubleCount: (state) => state.count * 2,
  },
});

    • Vuex：
      基于 state/mutations/actions/getters 的模块化设计。

      // store/index.js
import { createStore } from 'vuex';

export default createStore({
  state: { count: 0 },
  mutations: {
    increment(state) { state.count++; }
  },
  actions: {
    asyncIncrement({ commit }) { commit('increment'); }
  },
  getters: {
    doubleCount: (state) => state.count * 2,
  },
});

    # (2) 状态修改方式

    操作 Pinia Vuex
    读取状态 直接访问 store.count this.$store.state.count
    同步修改 直接赋值或调用 action 必须通过 commit 调用 mutation
    异步操作 直接在 action 中处理 需在 action 中调用 mutation

    Pinia 示例（更灵活）：

    const counterStore = useCounterStore();
counterStore.count++;          // 直接修改（需开启 devtools 记录）
counterStore.increment();      // 通过 action 修改

    Vuex 示例（严格模式限制）：

    this.$store.commit('increment');  // 必须通过 mutation
this.$store.dispatch('asyncIncrement');

    # 3. 模块化与组合

    特性 Pinia Vuex
    模块化 天然支持（每个 Store 独立文件） 需手动拆分 modules
    命名空间 自动隔离，无需配置 需设置 namespaced: true
    跨 Store 调用 直接导入其他 Store 使用 需通过 rootState/rootGetters

    Pinia 跨 Store 调用：

    import { useUserStore } from './user-store';

const counterStore = useCounterStore();
const userStore = useUserStore();

// 直接调用其他 Store 的 action
userStore.updateUserName('Alice');

    # 4. 开发体验对比

    # (1) TypeScript 支持

    • Pinia：
      类型推断开箱即用，无需额外配置。

      const counterStore = useCounterStore();
counterStore.count; // 自动推断为 number

    • Vuex：
      需要手动定义类型，或依赖 @vuex/types。

      interface State {
  count: number;
}
export default createStore<State>({ ... });

    # (2) DevTools 集成

    • Pinia：
      支持 Vue DevTools，自动记录状态变化和 actions。
    • Vuex：
      同样支持 DevTools，但需严格遵循 mutation 流程。

    # 5. 性能与体积

    指标 Pinia Vuex
    打包体积 约 1KB (gzipped) 约 10KB (gzipped)
    响应式系统 基于 Vue 3 的 reactive 基于 Vue 2 的 Vue.observable
    性能影响 更低（无嵌套响应式转换） 略高（需处理模块嵌套）

    # 7. 何时选择 Pinia 或 Vuex？

    # 选择 Pinia 的场景

    • 新项目基于 Vue 3 + TypeScript。
    • 希望更简洁的 API 和更小的体积。
    • 需要灵活的模块组合和直接状态修改。

    # 选择 Vuex 的场景

    • 维护现有的 Vue 2 项目。
    • 需要严格的 mutation 流程（适合大型团队协作）。
    • 依赖 Vuex 的插件生态（如持久化插件 vuex-persistedstate）。

    # 附录：代码片段对比

    # Pinia 的 Store

    // stores/counter.js
import { defineStore } from 'pinia';

export const useCounterStore = defineStore('counter', {
  state: () => ({ count: 0 }),
  actions: {
    increment() { this.count++; }
  },
  getters: {
    double: (state) => state.count * 2,
  },
});

// 组件中使用
const counter = useCounterStore();
counter.count++; // 直接修改

    # Vuex 的 Store

    // store/index.js
import { createStore } from 'vuex';

export default createStore({
  state: { count: 0 },
  mutations: {
    increment(state) { state.count++; }
  },
  actions: {
    asyncIncrement({ commit }) { commit('increment'); }
  },
  getters: {
    double: (state) => state.count * 2,
  },
});

// 组件中使用
this.$store.commit('increment'); // 必须通过 mutation
