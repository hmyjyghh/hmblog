    # 大模型实战用到的常用方法

    # 常用函数

    # 1. AutoTokenizer.from_pretrained()

    根据传的模型，自动加载和实例化与预训练模型对应的分词器（Tokenizer），无需手动指定具体的分词器类。

    eg:

    # 自动选择正确的分词器
tokenizer = AutoTokenizer.from_pretrained("bert-base-cased")        # → BertTokenizer
tokenizer = AutoTokenizer.from_pretrained("gpt2")                   # → GPT2Tokenizer  
tokenizer = AutoTokenizer.from_pretrained("t5-small")               # → T5Tokenizer
tokenizer = AutoTokenizer.from_pretrained("facebook/bart-large")    # → BartTokenizer

    # 2. BERT tokenizer输出的两个特殊标记。

    [CLS] 和 [SEP] 是BERT模型中的特殊标记，它们有非常重要的功能：

    [CLS] - Classification Token（分类标记）： 总是出现在序列的开头

    [SEP] - Separator Token（分隔标记）： 出现在序列的末尾

    # 实战步骤

    1. 首先引入torch.utils.data 中的DataSet类，构建自定义的数据集类，按照Pytorch的规范，需要实现__getitem__和__len__方法。
    2. 构建数据集，并使用DataLoader进行数据加载，设置好batch_size、shuffle等参数。
    3. 构建模型，并使用预训练的模型参数进行初始化。
    4. 定义损失函数和优化器。
    import torch
from torch.utils.data import Dataset, DataLoader
from transformers import BertTokenizer, BertForSequenceClassification

# 自定义数据集类
class MyDataset(Dataset):
  def __init__(self, data):
    self.data = data
    self.tokenizer = BertTokenizer.from_pretrained('bert-base-uncased')

  def __getitem__(self, index):
    text = self.data[index]['text']
    label = self.data[index]['label']
    encoding = self.tokenizer(text, truncation=True, padding='max_length', max_length=512, return_tensors='pt')
    1. 数据预处理

    使用DataLoader 库来按 batch 加载数据， 并使用PyTorch的DataLoader类来创建一个数据加载器。

    from torch.utils.data import DataLoader

# 创建数据加载器
train_loader = DataLoader(train_dataset, batch_size=32, shuffle=True)
    1. 模型构建

    使用预训练的BERT模型作为基础模型，并使用PyTorch的nn.Module类来构建自定义的模型。在模型中，我们只需要定义输入层和输出层，中间层使用预训练的BERT模型。

    # 模型的评价指标 BLEU

    • BLEU（Bilingual Evaluation Understudy）是一种用于评估机器翻译质量的指标。BLEU 的核心思想是，机器翻译的输出越接近人工翻译的结果，其质量就越高。
    from sacrebleu.metrics import BLEU

predictions = [
  "This plugin lets you translate web pages between several languages automatically."
]
bad_predictions_1 = ["This This This This"]
bad_predictions_2 = ["This plugin"]
references = [
  [
    "This plugin allows you to automatically translate web pages between several languages."
  ]
]

bleu = BLEU()
print(bleu.corpus_score(predictions, references).score)
print(bleu.corpus_score(bad_predictions_1, references).score)
print(bleu.corpus_score(bad_predictions_2, references).score)

1.683602693167689
0.0

BLEU 值的范围从 0 到 100，越高越好。

    # 什么是生成式问答？

    是的，这种实现写法属于生成式问答模型，主要体现在以下几个关键特征上：

    1. 模型选择：使用的T5ForConditionalGeneration是专为生成任务设计的模型，其核心能力是根据输入文本生成全新的输出序列，而非从给定文本中抽取片段。

    2. 输出方式：模型通过.generate()方法直接生成答案文本，这个答案是模型根据上下文和问题“创造”出来的，可能包含原文中没有的词汇组合（尽管在问答任务中通常会贴合原文）。

    3. 与抽取式的区别：

    • 抽取式问答（如BERT用于SQuAD任务）的输出是答案在原文中的起始和结束位置，答案必然是原文的子串
    • 而这段代码中，模型输出的是独立生成的文本序列，理论上可以超越原文范围（实际任务中会受训练数据约束）
    1. 输入格式：将问题和上下文拼接为"question: ... context: ..."的形式，这是典型的生成式问答提示（prompt）设计，模型需要理解这种指令并生成符合要求的答案。

    因此，这种实现完全符合生成式问答模型的定义——通过生成全新文本的方式来回答问题，而不是从上下文里抽取现成的答案片段。

    # Transformer 命名实体任务实战

    word_ids() 是 Hugging Face Tokenizer 中一个非常重要的方法，它的作用是 追踪分词后的每个 token 对应原始句子中的哪个单词。

    # 核心作用：建立分词前后的映射关系

    当句子被 tokenizer 分割成多个 subword tokens 时，word_ids() 可以帮助我们理解：

    • 哪些 tokens 属于同一个原始单词
    • 每个 token 在原始句子中的位置信息

    # 代码示例与解析

    from transformers import AutoTokenizer

sentence = '海钓比赛地点在厦门与金门之间的海域。'
tokenizer = AutoTokenizer.from_pretrained("bert-base-chinese")

# 分词并查看结果
tokens = tokenizer(sentence)
print("Tokens:", tokenizer.convert_ids_to_tokens(tokens['input_ids']))
print("Word IDs:", tokens.word_ids())

# 输出结果：
# Tokens: ['[CLS]', '海', '钓', '比', '赛', '地', '点', '在', '厦', '门', '与', '金', '门', '之', '间', '的', '海', '域', '。', '[SEP]']
# Word IDs: [None, 0, 0, 1, 1, 2, 2, 3, 4, 4, 5, 6, 6, 7, 7, 8, 9, 9, 10, None]

    # 结果解析

    Token Word ID 说明
    [CLS] None 特殊token，不属于任何单词
    海 0 属于第0个单词 "海钓"
    钓 0 属于第0个单词 "海钓"
    比 1 属于第1个单词 "比赛"
    赛 1 属于第1个单词 "比赛"
    地 2 属于第2个单词 "地点"
    点 2 属于第2个单词 "地点"
    在 3 属于第3个单词 "在"
    厦 4 属于第4个单词 "厦门"
    门 4 属于第4个单词 "厦门"
    ... ... ...
    [SEP] None 特殊token，不属于任何单词

    # 实际应用场景

    # 1. 命名实体识别（NER）

    # 假设我们有单词级别的标签
word_labels = ["O", "O", "O", "O", "B-LOC", "O", "O", "O"]

# 将单词标签映射到token级别
word_ids = tokens.word_ids()
token_labels = []

for word_id in word_ids:
    if word_id is None:
        token_labels.append("O")  # 特殊token
    else:
        token_labels.append(word_labels[word_id])

print("Token labels:", token_labels)

    # 2. 对齐文本和分词结果

    # 重建原始单词与tokens的对应关系
from collections import defaultdict

word_to_tokens = defaultdict(list)
tokens_list = tokenizer.convert_ids_to_tokens(tokens['input_ids'])
word_ids_list = tokens.word_ids()

for i, (token, word_id) in enumerate(zip(tokens_list, word_ids_list)):
    if word_id is not None:
        word_to_tokens[word_id].append((i, token))

print("单词到tokens的映射:")
for word_id, token_info in word_to_tokens.items():
    original_word = sentence.split()[word_id] if word_id < len(sentence.split()) else "N/A"
    print(f"单词{word_id}({original_word}): {token_info}")

    # 3. 处理英文句子（更直观）

    english_sentence = "I love machine learning!"
tokenizer_en = AutoTokenizer.from_pretrained("bert-base-uncased")
tokens_en = tokenizer_en(english_sentence)

print("English tokens:", tokenizer_en.convert_ids_to_tokens(tokens_en['input_ids']))
print("English word IDs:", tokens_en.word_ids())

# 输出：
# English tokens: ['[CLS]', 'i', 'love', 'machine', 'learning', '!', '[SEP]']
# English word IDs: [None, 0, 1, 2, 2, 3, None]
# 注意：'machine learning' 被分成两个tokens，但属于同一个单词索引(2)

    # 重要特性

    1. None 值：特殊 tokens ([CLS], [SEP], [PAD]等) 的 word_id 为 None
    2. 相同数字：属于同一个原始单词的所有 subwords 有相同的 word_id
    3. 连续编号：word_id 从 0 开始按原始单词顺序编号

    # 在训练中的实际用途

    def align_labels_with_tokens(labels, word_ids):
    """将单词级标签对齐到token级"""
    new_labels = []
    current_word = None
    
    for word_id in word_ids:
        if word_id is None:
            new_labels.append(-100)  # 忽略特殊token（PyTorch中常用-100）
        elif word_id != current_word:
            # 新单词的开始
            new_labels.append(labels[word_id])
            current_word = word_id
        else:
            # 同一单词的后续subword（如##ing）
            new_labels.append(-100)  # 或者使用相同的标签
    
    return new_labels

    总结：word_ids() 是处理 分词对齐问题 的关键工具，特别在序列标注任务（NER、词性标注等）中不可或缺。
