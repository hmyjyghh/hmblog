    # 大模型实战用到的常用方法

    # 常用函数

    # 1. AutoTokenizer.from_pretrained()

    根据传的模型，自动加载和实例化与预训练模型对应的分词器（Tokenizer），无需手动指定具体的分词器类。

    eg:

    # 自动选择正确的分词器
tokenizer = AutoTokenizer.from_pretrained("bert-base-cased")        # → BertTokenizer
tokenizer = AutoTokenizer.from_pretrained("gpt2")                   # → GPT2Tokenizer  
tokenizer = AutoTokenizer.from_pretrained("t5-small")               # → T5Tokenizer
tokenizer = AutoTokenizer.from_pretrained("facebook/bart-large")    # → BartTokenizer

    # 2. BERT tokenizer输出的两个特殊标记。

    [CLS] 和 [SEP] 是BERT模型中的特殊标记，它们有非常重要的功能：

    [CLS] - Classification Token（分类标记）： 总是出现在序列的开头

    [SEP] - Separator Token（分隔标记）： 出现在序列的末尾

    # 实战步骤

    1. 首先引入torch.utils.data 中的DataSet类，构建自定义的数据集类，按照Pytorch的规范，需要实现__getitem__和__len__方法。
    2. 构建数据集，并使用DataLoader进行数据加载，设置好batch_size、shuffle等参数。
    3. 构建模型，并使用预训练的模型参数进行初始化。
    4. 定义损失函数和优化器。
    import torch
from torch.utils.data import Dataset, DataLoader
from transformers import BertTokenizer, BertForSequenceClassification

# 自定义数据集类
class MyDataset(Dataset):
  def __init__(self, data):
    self.data = data
    self.tokenizer = BertTokenizer.from_pretrained('bert-base-uncased')

  def __getitem__(self, index):
    text = self.data[index]['text']
    label = self.data[index]['label']
    encoding = self.tokenizer(text, truncation=True, padding='max_length', max_length=512, return_tensors='pt')
    1. 数据预处理

    使用DataLoader 库来按 batch 加载数据， 并使用PyTorch的DataLoader类来创建一个数据加载器。

    from torch.utils.data import DataLoader

# 创建数据加载器
train_loader = DataLoader(train_dataset, batch_size=32, shuffle=True)
    1. 模型构建

    使用预训练的BERT模型作为基础模型，并使用PyTorch的nn.Module类来构建自定义的模型。在模型中，我们只需要定义输入层和输出层，中间层使用预训练的BERT模型。

    # 模型的评价指标 BLEU

    • BLEU（Bilingual Evaluation Understudy）是一种用于评估机器翻译质量的指标。BLEU 的核心思想是，机器翻译的输出越接近人工翻译的结果，其质量就越高。
    from sacrebleu.metrics import BLEU

predictions = [
  "This plugin lets you translate web pages between several languages automatically."
]
bad_predictions_1 = ["This This This This"]
bad_predictions_2 = ["This plugin"]
references = [
  [
    "This plugin allows you to automatically translate web pages between several languages."
  ]
]

bleu = BLEU()
print(bleu.corpus_score(predictions, references).score)
print(bleu.corpus_score(bad_predictions_1, references).score)
print(bleu.corpus_score(bad_predictions_2, references).score)

    # 什么是生成式问答？

    是的，这种实现写法属于生成式问答模型，主要体现在以下几个关键特征上：

    1. 模型选择：使用的T5ForConditionalGeneration是专为生成任务设计的模型，其核心能力是根据输入文本生成全新的输出序列，而非从给定文本中抽取片段。

    2. 输出方式：模型通过.generate()方法直接生成答案文本，这个答案是模型根据上下文和问题“创造”出来的，可能包含原文中没有的词汇组合（尽管在问答任务中通常会贴合原文）。

    3. 与抽取式的区别：

    • 抽取式问答（如BERT用于SQuAD任务）的输出是答案在原文中的起始和结束位置，答案必然是原文的子串
    • 而这段代码中，模型输出的是独立生成的文本序列，理论上可以超越原文范围（实际任务中会受训练数据约束）
    1. 输入格式：将问题和上下文拼接为"question: ... context: ..."的形式，这是典型的生成式问答提示（prompt）设计，模型需要理解这种指令并生成符合要求的答案。

    因此，这种实现完全符合生成式问答模型的定义——通过生成全新文本的方式来回答问题，而不是从上下文里抽取现成的答案片段。
