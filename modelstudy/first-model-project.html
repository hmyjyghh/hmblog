    BM25Retriever 是 LangChain 社区提供的基于 BM25 算法 的检索器，适用于无向量依赖的文本匹配场景（无需嵌入模型），核心是通过词频、文档长度等统计特征计算文本相似度，快速召回相关文档。

    # 一、安装依赖

    首先确保安装 langchain-community（BM25Retriever 所在包），若需处理中文可额外安装分词库（如 jieba）：

    pip install langchain-community  # 核心依赖
pip install jieba  # 中文分词（可选，默认英文分词）

    # 二、核心概念

    • BM25 算法：一种经典的信息检索排序算法，优化了 TF-IDF 对长文档的惩罚问题，通过 k1（词频饱和度）、b（文档长度归一化系数）等参数调节匹配效果。
    • 文档格式：输入需为 List[Document]（LangChain 的文档类，包含 page_content 文本内容和 metadata 元数据）。
    • 分词逻辑：默认使用英文分词（按空格分割），中文需自定义分词函数。

    # 三、基础用法（英文场景）

    # 步骤 1：导入依赖 & 准备文档

    from langchain_community.retrievers import BM25Retriever
from langchain_core.documents import Document

# 1. 准备文档（List[Document] 格式，必须包含 page_content）
documents = [
    Document(
        page_content="LangChain is a framework for building LLM applications",
        metadata={"source": "langchain-docs"}
    ),
    Document(
        page_content="BM25Retriever is a statistical retriever in LangChain",
        metadata={"source": "langchain-retrievers"}
    ),
    Document(
        page_content="LLMs are powerful language models like GPT-4",
        metadata={"source": "llm-intro"}
    ),
    Document(
        page_content="LangChain supports various retrievers: BM25, VectorStore, etc.",
        metadata={"source": "langchain-features"}
    )
]

    # 步骤 2：初始化 BM25Retriever 并添加文档

    # 2. 初始化检索器（默认英文分词）
retriever = BM25Retriever.from_documents(documents)

# 或分步初始化（先创建实例，再添加文档）
# retriever = BM25Retriever()
# retriever.add_documents(documents)

    # 步骤 3：检索相关文档

    使用 get_relevant_documents(query) 方法，返回按相似度排序的文档列表：

    # 3. 检索查询
query = "What retrievers does LangChain support?"
relevant_docs = retriever.get_relevant_documents(query)

# 4. 打印结果
for i, doc in enumerate(relevant_docs, 1):
    print(f"\n=== 相关文档 {i} ===")
    print(f"文本内容：{doc.page_content}")
    print(f"元数据：{doc.metadata}")

    输出示例（按相似度降序）：

    === 相关文档 1 ===
文本内容：LangChain supports various retrievers: BM25, VectorStore, etc.
元数据：{'source': 'langchain-features'}

=== 相关文档 2 ===
文本内容：BM25Retriever is a statistical retriever in LangChain
元数据：{'source': 'langchain-retrievers'}

=== 相关文档 3 ===
文本内容：LangChain is a framework for building LLM applications
元数据：{'source': 'langchain-docs'}

    # 四、中文场景适配（自定义分词）

    BM25Retriever 默认按空格分词，中文需替换为中文分词函数（如 jieba），通过 preprocess_func 参数配置：

    import jieba
from langchain_community.retrievers import BM25Retriever
from langchain_core.documents import Document

# 1. 中文文档
chinese_docs = [
    Document(page_content="LangChain 是一个用于构建大语言模型应用的框架", metadata={"来源": "LangChain 官方文档"}),
    Document(page_content="BM25Retriever 是 LangChain 中的统计型检索器，无需嵌入模型", metadata={"来源": "LangChain 社区"}),
    Document(page_content="大语言模型（LLM）如 GPT-4、文心一言，具备强大的文本生成能力", metadata={"来源": "AI 百科"}),
    Document(page_content="LangChain 支持多种检索器：BM25、向量数据库检索器等", metadata={"来源": "LangChain 特性文档"})
]

# 2. 定义中文分词函数（输入字符串，返回分词列表）
def chinese_tokenizer(text):
    return list(jieba.cut(text))  # jieba 分词，返回列表

# 3. 初始化检索器（指定 preprocess_func 为中文分词）
chinese_retriever = BM25Retriever.from_documents(
    chinese_docs,
    preprocess_func=chinese_tokenizer  # 关键：替换分词逻辑
)

# 4. 中文查询检索
query = "LangChain 支持哪些检索器？"
relevant_docs = chinese_retriever.get_relevant_documents(query)

# 打印结果
for i, doc in enumerate(relevant_docs, 1):
    print(f"\n=== 相关文档 {i} ===")
    print(f"文本内容：{doc.page_content}")
    print(f"元数据：{doc.metadata}")

    # 五、核心参数配置（优化检索效果）

    BM25Retriever 的初始化参数可调节匹配逻辑，核心参数如下：

    参数名 作用说明 默认值 推荐范围
    k1 词频饱和度系数（控制词频对相似度的影响，越大词频权重越高） 1.5 1.0 ~ 2.0
    b 文档长度归一化系数（控制文档长度对相似度的惩罚，b=0不惩罚，b=1最大惩罚） 0.75 0.5 ~ 0.8
    epsilon 平滑参数（避免零概率） 0.25 0.1 ~ 0.5
    preprocess_func 分词函数（输入文本字符串，返回分词列表） 英文分词 自定义中文分词
    max_documents 每次检索返回的最大文档数 4 根据需求调整

    # 示例：调节参数优化效果

    # 初始化时指定参数
retriever = BM25Retriever(
    k1=1.2,  # 降低词频权重，避免过度依赖高频词
    b=0.6,   # 减轻文档长度惩罚
    epsilon=0.3,
    preprocess_func=chinese_tokenizer,
    max_documents=5  # 最多返回5篇相关文档
)
retriever.add_documents(chinese_docs)  # 手动添加文档

    # 六、进阶用法

    # 1. 动态添加/删除文档

    支持后续补充文档或清空文档库：

    # 添加新文档
new_doc = Document(page_content="LangChain 还支持自定义检索器扩展", metadata={"来源": "扩展文档"})
retriever.add_documents([new_doc])  # 接收 List[Document]

# 清空所有文档
retriever.clear()

    # 2. 结合 LangChain 管道（Chain）使用

    可与 RetrievalQA 等链结合，快速搭建问答系统：

    from langchain.chains import RetrievalQA
from langchain_community.llms import OpenAI  # 或其他LLM（如本地化模型）

# 1. 初始化LLM（示例用OpenAI，可替换为本地化模型如通义千问）
llm = OpenAI(api_key="your-api-key")

# 2. 构建问答链
qa_chain = RetrievalQA.from_chain_type(
    llm=llm,
    chain_type="stuff",  # 直接将文档内容传入LLM
    retriever=retriever,  # 关联BM25Retriever
    return_source_documents=True  # 返回检索到的原始文档
)

# 3. 提问
result = qa_chain.invoke("LangChain 支持哪些检索器？")
print("回答：", result["result"])
print("来源文档：", [doc.page_content for doc in result["source_documents"]])

    # 3. 批量检索与相似度分数

    默认 get_relevant_documents 仅返回文档，若需获取相似度分数，可通过 similarity_search_with_score 方法：

    # 批量检索（支持多个查询）
queries = ["LangChain 框架", "大语言模型应用"]
for q in queries:
    docs_with_scores = retriever.similarity_search_with_score(q, k=2)  # k=返回Top2
    print(f"\n查询：{q}")
    for doc, score in docs_with_scores:
        print(f"  文档：{doc.page_content}，相似度分数：{score:.4f}")

    # 七、适用场景与注意事项

    # 适用场景

    • 无嵌入模型（如 BERT）时的快速检索；
    • 英文/中文短文本匹配（文档长度不宜过长，建议单文档≤500字）；
    • 对检索速度要求高，无需复杂语义理解的场景。

    # 注意事项

    • 不支持语义理解：仅基于词频、字面匹配，无法处理同义词（如“检索器”和“搜索工具”视为不同词）；
    • 长文档效果差：文档过长会导致词频分散，建议先拆分长文档为短片段；
    • 中文需分词：必须自定义中文分词函数（如 jieba、THULAC），否则按单个汉字分词，效果极差。

    # 八、常见问题排查

    1. 中文检索无结果/效果差：

      • 检查是否配置了 preprocess_func 中文分词；
      • 确认分词函数返回的是列表（如 list(jieba.cut(text))），而非字符串。

    2. 检索返回文档数不足：

      • 调整 max_documents 参数（默认4）；
      • 检查文档总数是否少于 k（similarity_search_with_score 的 k 参数）。

    3. 词频过高的词影响结果：

      • 降低 k1 参数（如从1.5调整为1.0）；
      • 预处理文档，过滤停用词（如“的、是、在”等无意义词）。

    通过以上用法，可快速上手 BM25Retriever 并适配中英文场景，根据实际需求调节参数优化检索效果。
