    向量数据库

    寒梦

    # 向量数据库

    以向量的形式存储和检索数据，向量是复杂对象(如文本，图像或音频)的数值表示

    • 它是一种专门设计，用于处理高维数据的数据库，其中每个条目表示为多维空间中的一个向量。
    • 向量数据库的核心优势在于，能够高效地存储，索引 和 搜索向量
    • 有各种相似性搜索算法，特别适合推荐系统(eg: 相关性推荐)

    # 1. 前置概念

    • 向量是具有大小和方向的数学结构
    • 可以将事物的特征用向量来表示

    # 2. 向量数据库的核心思想

    将文本转为向量，然后将向量存储到数据库中

    • 当用户输入问题时，将问题转换为向量
    • 然后在数据库中搜索最相似的向量和匹配最相似的几个上下文
    • 最后将文本返回给用户

    # 3. 向量数据库 vs 传统数据库

    • 向量数据库与传统数据库的区别在于: 它们的应用领域和处理数据的方式。

    # 两者的对比

    对比维度 向量数据库 传统数据库
    数据表示 数据点以具有固定维度的向量形式表示 数据以表格、行和列的形式组织
    数据类型 多维向量、embeddings 结构化数据（文本、数字、日期）
    查询方式 近似最近邻（ANN）搜索、语义搜索 精确查询、关键词搜索、关系型联接
    应用场景 推荐系统、欺诈检测、异常检测、聊天机器人 银行业务系统、库存管理、客户关系管理（CRM）
    优势 高效的相似性搜索，处理复杂数据，具备良好的扩展性 强大的数据完整性，支持ACID特性

    # 核心点分析

    • 数据类型不同
    • 查询方式： 近似查询，精确查询

    # 4. 向量数据库的工作流程

    向量数据库通过近似最近邻(ANN)搜索来查找相似的向量, 从而实现语义信息检索。

    # 3个步骤

    • 索引(Indexing): 向量数据库将图像、文本、音频或视频内容转换为向量,并使用PQ、LSH或HNSW 等算法对这些向量进行索引。
      • 这些算法有助于高效处理和存储大量的向量数据, 便于后续的快速检索。
    • 查询(Querying): 不同类型的查询(如文本、图像查询)会先被转换为向量, 然后与已经索引的向量进行最近邻比较, 通过最近邻算法找到最相似的结果。
    • 后处理(Post-processing): 在某些情况下, 向量数据库会从数据集中检索到最近邻向量, 并对它们进行后续处理以返回最终结果。
      • 这个过程可能包括: 使用不同的相似度度量方法, 对最近邻结果进行重新排序, 以提升检索结果的准确性和相关性。

    # 5. 向量检索的基本原理

    核心原理：基于索引和查询操作

    以PQ 量化，乘积量化为例

    1. 向量数据库将所有输入的向量转化为指定的维度，并通过聚类算法将它们分组，每个聚类中心都可以视为一个圆形向量, 并用于构建最终的索引。
    2. 在查询阶段呢，向量数据库 通过计算：输入向量和所有圆形向量之间的距离，快速找到最相似的聚类中心，从而完成向量的检索。

    总的来说，其实就是向量数据库

    • 在利用量化的技术或其他近似邻搜索算法来减少搜索空间，提高检索速度，
    • 同时尽可能保证检索的准确性。

    # 通俗易懂的解释

    就是找到和查询向量最接近的那个聚类中心，缩小搜索范围，再去检索

    # 6. BM25、TF-IDF，排序算法

    • 本质还是基于文本的精确匹配
    • 但对于语义搜索功能就非常弱

    # 7. Vector Embedding

    Vector Embedding 是由 AI 模型（例如大型语言模型 LLM）生成的，它会根据不同的算法生成高维度的向量数据，代表着数据的不同特征，这些特征代表了数据的不同维度。

    例如这份文档是客服培训资料或者操作手册，我们可以先将这份文档的所有内容转化成向量（这个过程称之为 Vector Embedding）

    • 例如，对于文本，这些特征可能包括词汇、语法、语义、情感、情绪、主题、上下文等。对于音频，这些特征可能包括音调、节奏、音高、音色、音量、语音、音乐等。

    # 8. 特征和向量

    1. 如何判断相似度?
    • 可以通过比较向量之间的距离来判断它们的相似度

    # 9. 相似性搜索(Similarity Search)

    • 高效的搜索算法有很多，其主要思想是通过两种方式提高搜索效率：

    1. 减少向量大小：通过降维或减少表示向量值的长度。

    2. 缩小搜索范围：可以通过聚类或将向量组织成基于树形、图形结构来实现，并限制搜索范围仅在最接近的簇中进行，或者通过最相似的分支进行过滤。

    # 10. 近似最近邻算法(Approximate Nearest Neighbor)

    • 包含暴力搜索和聚类算法

    常见的聚类算法有 K-Means

    k-means 算法的基本步骤

    1. 选择k个初始聚类中心。
    2. 将每个数据点分配到最近的聚类中心。
    3. 计算每个聚类的新中心。
    4. 重复步骤 2 和 3，直到聚类中心不再改变或达到最大迭代次数(趋于稳定)。

    # 11. PQ 算法

    出现的原因是： 在大规模数据集中，聚类算法最大的问题在于内存占用太大。

    1. 除了要保存每个向量的坐标
    2. 还需要维护聚类中心和每个向量的聚类中心索引，这也会占用大量的内存。

    对于问题1： 可以通过量化 (Quantization) 的方式解决，也就是常见的有损压缩。

    降维：

    • 比如将 128 维的向量分为 8 个 16 维的向量，
    • 然后在 8 个 16 维的子向量上分别进行聚类，因为 16 维的子向量大概只需要 256 个聚类中心就能得到还不错的量化结果，所以就可以将码本的大小从 2^64 降低到 8 * 256 = 2048 个聚类中心，从而降低内存开销。

    这也就是乘积量化（Product Quantization）的原理。

    • 使用 PQ 算法，可以显著的减少内存的开销，同时加快搜索的速度，
    • 它唯一的问题是搜索的质量会有所下降，但就像我们刚才所讲，所有算法都是在内存、速度和质量上做一个权衡。

    # 12. HNSW

    Hierarchical Navigable Small Worlds: 分层导航的小世界

    • 除了聚类以外，也可以通过构建树或者构建图的方式来实现近似最近邻搜索。
    • 这种方法的基本思想是每次将向量加到数据库中的时候，就先找到与它最相邻的向量，然后将它们连接起来，这样就构成了一个图。
    • 当需要搜索的时候，就可以从图中的某个节点开始，不断的进行最相邻搜索和最短路径计算，直到找到最相似的向量。

    HNSW 算法是一种经典的空间换时间的算法，它的搜索质量和搜索速度都比较高，但是它的内存开销也比较大，因为不仅需要将所有的向量都存储在内存中。还需要维护一个图的结构，也同样需要存储。所以这类算法需要根据实际的场景来选择。

    # 13. 局部敏感哈希算法(LSH)

    • 或者叫位置敏感哈希函数

    • 局部敏感哈希（Locality Sensitive Hashing）也是一种使用近似最近邻搜索的索引技术。

    • 它的特点是快速，同时仍然提供一个近似、非穷举的结果。

    • LSH 使用一组哈希函数将相似向量映射到“桶”中，从而使相似向量具有相同的哈希值。

    • 这样，就可以通过比较哈希值来判断向量之间的相似度。

    # 14. 相似性测量 (Similarity Measurement)

    1. 欧几里得距离（Euclidean Distance）
    2. 余弦相似度（Cosine Similarity）
    3. 点积相似度 (Dot product Similarity)
    • 余弦距离，越⼤越相似
    • 欧式距离，越⼩越相似

    在您的RAG系统中，文本嵌入通常使用余弦相似度，因为它更关注语义方向而非向量长度。

    # 15. 过滤 (Filtering)

    在实际的业务场景中，往往不需要在整个向量数据库中进行相似性搜索，而是通过部分的业务字段进行过滤再进行查询。

    • 所以存储在数据库的向量往往还需要包含元数据，例如用户 ID、文档 ID 等信息。 -`这样就可以在搜索的时候，根据元数据来过滤搜索结果，从而得到最终的结果。

    # 16. 如何选型向量数据库?

    # Milvus

    Milvus的关键特性包括：

    • 支持海量数据的快速检索
    • 灵活的数据更新和扩展的功能, 具有一个高可用的架构
    • 生态做的很好，可以无缝去集成这种深度学习的框架，包括模型，比如常见的一些向量模型，ranker模型，都能用Milvus来做集成
    • 支持混合检索
    • 检索效率高，毫秒级搜索万亿级向量数据集

    # Chroma

    Chroma的关键特性包括:

    • 功能丰富：⽀持查询、过滤、密度估计等多种功能
    • 即将添加的语⾔链（LangChain）、LlamaIndex等更多功能
    • 相同的API可以在Python笔记本中运⾏，也可以扩展到集群，⽤于开发、测试和⽣产

    # Faiss

    Faiss的关键特性包括：

    • 不仅返回最近的邻居，还返回第⼆近、第三近和第k近的邻居

    • 可以同时搜索多个向量，⽽不仅仅是单个向量（批量处理）

    • 使⽤最⼤内积搜索⽽不是最⼩欧⼏⾥得搜索

    • 也⽀持其他距离度量，但程度较低。

    • 返回查询位置附近指定半径内的所有元素（范围搜索）

    • 可以将索引存储在磁盘上，⽽不仅仅是RAM中

    • 如何选型？

    需要根据项⽬的具体需求、团队的技术背景和资源情况来综合评估。

    # 17. 项目实战中的用法

    # Milvus 向量数据库

    from pymilvus import AnnSearchRequest

    在 PyMilvus 中，AnnSearchRequest 是 向量近邻搜索（Approximate Nearest Neighbor Search）的请求封装类，核心作用是：将单次向量搜索的所有参数（查询向量、搜索集合、搜索参数等）统一封装成一个请求对象，支持两种核心使用场景：

    # 一、核心作用：封装单次搜索的完整参数

    向量搜索需要多个关键参数（如查询向量、目标集合、距离度量方式、TopK 等），AnnSearchRequest 把这些参数整合为一个结构化对象，避免参数分散传递导致的混乱，同时让代码更易维护。

    # 必须/常用参数说明（初始化时指定）：

    参数名 类型/说明
    data 查询向量数据（如 [[1.2, 3.4, ...]]，支持单向量或多向量批量查询）
    collection_name 要搜索的 Milvus 集合名称（字符串）
    param 搜索配置字典（核心！如 {"metric_type": "L2", "params": {"nprobe": 10}}）
    limit 返回的TopK结果数（默认10，即返回与查询向量最相似的前K个结果）
    output_fields 需要返回的字段列表（默认只返回主键和距离，可指定其他字段如 ["text", "label"]）
    partition_names 要搜索的分区名称列表（可选，不指定则搜索全集合）
    expr 布尔过滤表达式（可选，如 label == 1，只搜索满足条件的数据）
    consistency_level 搜索的一致性级别（可选，如 ConsistencyLevel.STRONG，默认继承集合配置）

    # 场景 2：多集合 / 多参数联合搜索（核心价值场景）

    这是 AnnSearchRequest 最关键的用途：同时向多个集合（或同一集合的不同分区 / 不同搜索参数）发起搜索，再通过 RRFRanker 或 WeightedRanker 对所有结果进行融合排序。

    # 4. 执行联合搜索，用 RRFRanker 融合结果（基于召回率的融合排序）
ranker = RRFRanker(k=60)  # k是融合时参考的总结果数，默认60
results = utility.search_multi(
    requests=[req1, req2],  # 传入多个搜索请求
    ranker=ranker,          # 指定结果融合策略
    limit=20                # 最终返回的TopK结果
)

    # 总结

    AnnSearchRequest 的核心价值是 “结构化封装搜索参数” + “支持多集合联合搜索”：

    • 简单场景下：让单次搜索的参数更清晰；
    • 复杂场景下：实现多集合/多参数的联合搜索与结果融合，是 Milvus 中实现跨集合向量检索的关键类。
