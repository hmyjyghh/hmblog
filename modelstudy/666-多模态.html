    Vit Transformer (opens new window)

    仓库中涉及的 Vision Transformer (ViT) 实现涵盖了 PyTorch 和 TensorFlow 两个框架，核心是将Transformer架构应用于图像分类任务，遵循"图像分块→嵌入→Transformer编码→分类"的经典流程。以下从实现细节、核心组件和框架差异三方面分析：

    # 一、核心原理与整体结构

    所有实现均遵循ViT的原始设计思路：

    1. 图像分块（Patch Embedding）：将输入图像分割为固定大小的非重叠补丁（如16×16），通过线性投影将每个补丁转换为向量（嵌入向量）。
    2. 添加类别Token与位置嵌入：在补丁嵌入序列前拼接一个可学习的cls_token（用于最终分类），并叠加位置嵌入（编码补丁的空间位置信息）。
    3. Transformer编码器：由多个Transformer块组成，每个块包含多头自注意力（Multi-Head Self-Attention）和MLP，通过注意力机制捕捉补丁间的全局依赖。
    4. 分类头：取cls_token经过Transformer编码后的输出，通过全连接层得到分类结果。

    # 二、PyTorch实现（pytorch_classification下的vit_model.py）

    以pytorch_classification/vision_transformer/vit_model.py为例，核心组件如下：

    # 1. PatchEmbed（补丁嵌入）

    class PatchEmbed(nn.Module):
    def __init__(self, img_size=224, patch_size=16, in_c=3, embed_dim=768, norm_layer=None):
        self.proj = nn.Conv2d(in_c, embed_dim, kernel_size=patch_size, stride=patch_size)
        # 卷积层实现分块与投影：将每个16×16补丁映射为768维向量
    def forward(self, x):
        x = self.proj(x).flatten(2).transpose(1, 2)  # [B, C, H, W] → [B, num_patches, embed_dim]
    • 用卷积层Conv2d同时完成分块（步长=补丁大小）和线性投影，效率高于先分块再线性层。
    • 输出形状为[B, num_patches, embed_dim]，其中num_patches = (img_size/patch_size)²（如224×224图像→14×14=196个补丁）。

    # 2. Attention（多头自注意力）

    class Attention(nn.Module):
    def __init__(self, dim, num_heads=8, qkv_bias=False, attn_drop_ratio=0., proj_drop_ratio=0.):
        self.qkv = nn.Linear(dim, dim * 3, bias=qkv_bias)  # 一次性计算Q、K、V
        self.scale = (dim // num_heads) ** -0.5  # 缩放因子，避免注意力分数过大
    def forward(self, x):
        B, N, C = x.shape  # N = num_patches + 1（含cls_token）
        qkv = self.qkv(x).reshape(B, N, 3, self.num_heads, C//self.num_heads).permute(2, 0, 3, 1, 4)
        q, k, v = qkv[0], qkv[1], qkv[2]  # 拆分Q、K、V
        attn = (q @ k.transpose(-2, -1)) * self.scale  # 注意力分数：[B, heads, N, N]
        attn = attn.softmax(dim=-1)  # 归一化
        x = (attn @ v).transpose(1, 2).reshape(B, N, C)  # 加权求和并合并头
    • 多头注意力将输入向量拆分到多个头并行计算，捕捉不同子空间的特征。
    • 用Linear层一次性计算Q、K、V，再通过维度变换拆分，提升效率。

    # 3. Mlp（多层感知器）

    class Mlp(nn.Module):
    def __init__(self, in_features, hidden_features=None, act_layer=nn.GELU, drop=0.):
        self.fc1 = nn.Linear(in_features, hidden_features)  # 升维
        self.act = act_layer()  # GELU激活
        self.fc2 = nn.Linear(hidden_features, in_features)  # 降维
    def forward(self, x):
        x = self.fc1(x) → self.act(x) → self.drop(x) → self.fc2(x) → self.drop(x)
    • 两层线性层+GELU激活，隐藏层维度通常为输入的4倍（mlp_ratio=4），实现特征的非线性变换。

    # 4. Block（Transformer块）

    class Block(nn.Module):
    def __init__(self, dim, num_heads, mlp_ratio=4., drop_path_ratio=0.):
        self.norm1 = nn.LayerNorm(dim)  # 注意力前的层归一化
        self.attn = Attention(...)      # 多头注意力
        self.drop_path = DropPath(...)  # 随机深度（可选，防止过拟合）
        self.norm2 = nn.LayerNorm(dim)  # MLP前的层归一化
        self.mlp = Mlp(...)             # MLP模块
    def forward(self, x):
        x = x + self.drop_path(self.attn(self.norm1(x)))  # 残差+注意力
        x = x + self.drop_path(self.mlp(self.norm2(x)))    # 残差+MLP
    • 采用“预归一化”（Norm在前，Attention/MLP在后）设计，更稳定训练。
    • DropPath（随机深度）随训练过程随机丢弃部分块，增强泛化能力。

    # 5. VisionTransformer（主模型）

    class VisionTransformer(nn.Module):
    def __init__(self, img_size=224, patch_size=16, embed_dim=768, depth=12, num_heads=12, ...):
        self.patch_embed = PatchEmbed(...)  # 补丁嵌入
        self.cls_token = nn.Parameter(torch.zeros(1, 1, embed_dim))  # 类别Token
        self.pos_embed = nn.Parameter(torch.zeros(1, num_patches+1, embed_dim))  # 位置嵌入
        self.blocks = nn.Sequential(*[Block(...) for _ in range(depth)])  # Transformer块序列
        self.norm = nn.LayerNorm(embed_dim)  # 最终层归一化
        self.head = nn.Linear(embed_dim, num_classes)  # 分类头
    def forward(self, x):
        x = self.patch_embed(x)  # [B, num_patches, embed_dim]
        x = torch.cat([self.cls_token.expand(B, -1, -1), x], dim=1)  # 拼接cls_token
        x = x + self.pos_embed  # 叠加位置嵌入
        x = self.blocks(x)      # Transformer编码
        x = self.norm(x)        # 归一化
        x = self.head(x[:, 0])  # 取cls_token输出分类
    • 可配置参数丰富：图像大小、补丁大小、嵌入维度、Transformer深度、头数等，支持不同规模的模型（如ViT-Base：depth=12, num_heads=12）。
    • 支持“蒸馏Token”（distilled=True），适配DeiT等蒸馏版本模型。

    # 四、关键共性与设计亮点

    1. 模块化设计：各组件（PatchEmbed、Attention、Block）解耦，便于复用和修改（如替换不同的嵌入方式或注意力机制）。
    2. 兼容性：支持自定义数据集和超参数（如img_size、num_classes），可灵活适配不同任务。
    3. 与原始论文对齐：严格遵循ViT的核心设计（如位置嵌入为可学习参数、cls_token用于分类），同时加入工程优化（如卷积实现分块、随机深度）。

    # 总结

    仓库中的Vision Transformer实现完整复现了原始论文的核心逻辑，同时针对PyTorch和TensorFlow框架做了适配，代码结构清晰、可配置性强。PyTorch版本更注重模块化和训练稳定性（如随机深度），TensorFlow版本则贴合Keras的层式编程风格，两者均适合作为学习ViT架构或二次开发的基础模板。
