    在 PyMilvus 中，AnnSearchRequest 是 向量近邻搜索（Approximate Nearest Neighbor Search）的请求封装类，核心作用是：将单次向量搜索的所有参数（查询向量、搜索集合、搜索参数等）统一封装成一个请求对象，支持两种核心使用场景：

    # 一、核心作用：封装单次搜索的完整参数

    向量搜索需要多个关键参数（如查询向量、目标集合、距离度量方式、TopK 等），AnnSearchRequest 把这些参数整合为一个结构化对象，避免参数分散传递导致的混乱，同时让代码更易维护。

    # 必须/常用参数说明（初始化时指定）：

    参数名 类型/说明
    data 查询向量数据（如 [[1.2, 3.4, ...]]，支持单向量或多向量批量查询）
    collection_name 要搜索的 Milvus 集合名称（字符串）
    param 搜索配置字典（核心！如 {"metric_type": "L2", "params": {"nprobe": 10}}）
    limit 返回的TopK结果数（默认10，即返回与查询向量最相似的前K个结果）
    output_fields 需要返回的字段列表（默认只返回主键和距离，可指定其他字段如 ["text", "label"]）
    partition_names 要搜索的分区名称列表（可选，不指定则搜索全集合）
    expr 布尔过滤表达式（可选，如 label == 1，只搜索满足条件的数据）
    consistency_level 搜索的一致性级别（可选，如 ConsistencyLevel.STRONG，默认继承集合配置）

    # 二、两大核心使用场景

    # 场景1：单次简单搜索（替代 collection.search() 的直接调用）

    虽然可以直接用 Collection.search() 方法，但 AnnSearchRequest 能让参数更结构化，尤其适合参数较多的场景。

    示例代码：

    from pymilvus import AnnSearchRequest

# 1. 构造搜索请求（封装所有参数）
search_req = AnnSearchRequest(
    data=[[0.1, 0.2, 0.3, 0.4]],  # 查询向量（假设是4维向量）
    collection_name="my_collection",  # 目标集合
    param={
        "metric_type": "L2",  # 距离度量方式（L2欧氏距离，可选IP余弦距离等）
        "params": {"nprobe": 10}  # 搜索参数（nprobe越大，精度越高但速度越慢）
    },
    limit=20,  # 返回前20个最相似结果
    output_fields=["text", "label"],  # 除了主键和距离，还返回text和label字段
    expr="label == 0",  # 只搜索label=0的数据（过滤条件）
    consistency_level="STRONG"  # 强一致性搜索
)

# 2. 执行搜索（通过 connections 或 collection 调用）
collection = Collection("my_collection")
results = collection.search(requests=[search_req])  # 接收请求列表，支持单次多个请求

    # 场景2：多集合/多参数联合搜索（核心价值场景）

    这是 AnnSearchRequest 最关键的用途：同时向多个集合（或同一集合的不同分区/不同搜索参数）发起搜索，再通过 RRFRanker 或 WeightedRanker 对所有结果进行融合排序。

    比如：你有两个集合 collection_a（存储短文本向量）和 collection_b（存储长文本向量），想同时搜索两个集合，再合并结果返回。

    示例代码：

    from pymilvus import (
    connections, Collection, AnnSearchRequest, RRFRanker
)

# 1. 连接 Milvus
connections.connect("default", host="localhost", port="19530")

# 2. 加载两个目标集合
coll_a = Collection("short_text_vec")
coll_b = Collection("long_text_vec")
coll_a.load()
coll_b.load()

# 3. 构造两个搜索请求（分别对应两个集合，可配置不同参数）
req1 = AnnSearchRequest(
    data=[[0.1, 0.2, 0.3, 0.4]],  # 同一查询向量
    collection_name="short_text_vec",
    param={"metric_type": "IP", "params": {"nprobe": 8}},  # 余弦距离，nprobe=8
    limit=15,
    output_fields=["text"]
)

req2 = AnnSearchRequest(
    data=[[0.1, 0.2, 0.3, 0.4]],  # 同一查询向量
    collection_name="long_text_vec",
    param={"metric_type": "IP", "params": {"nprobe": 12}},  # 余弦距离，nprobe=12
    limit=15,
    output_fields=["text"]
)

# 4. 执行联合搜索，用 RRFRanker 融合结果（基于召回率的融合排序）
ranker = RRFRanker(k=60)  # k是融合时参考的总结果数，默认60
results = utility.search_multi(
    requests=[req1, req2],  # 传入多个搜索请求
    ranker=ranker,          # 指定结果融合策略
    limit=20                # 最终返回的TopK结果
)

# 5. 解析结果（融合了两个集合的相似结果，按融合后分数排序）
for hit in results[0]:  # results[0] 对应第一个查询向量的结果
    print(f"文本：{hit.entity.get('text')}, 融合后分数：{hit.score}, 来自集合：{hit.collection_name}")

    # 三、关键补充说明

    1. 与 collection.search() 的区别：

      • collection.search() 是单个集合的搜索方法，参数分散传递；
      • AnnSearchRequest 是请求封装，支持多集合联合搜索（需配合 utility.search_multi()），且参数更结构化。

    2. 结果融合器：

      • 多请求搜索时必须指定 ranker：
        • RRFRanker：基于召回率的融合（适合多集合互补搜索，避免结果重复）；
        • WeightedRanker：加权融合（可给不同请求设置权重，如 WeightedRanker(weights=[0.7, 0.3]) 表示第一个请求结果权重70%，第二个30%）。

    3. 批量查询支持：data 参数可传入多组向量（如 [[0.1,0.2], [0.3,0.4]]），此时返回结果会按向量顺序对应每组的搜索结果。

    # 总结

    AnnSearchRequest 的核心价值是 “结构化封装搜索参数” + “支持多集合联合搜索”：

    • 简单场景下：让单次搜索的参数更清晰；
    • 复杂场景下：实现多集合/多参数的联合搜索与结果融合，是 Milvus 中实现跨集合向量检索的关键类。
