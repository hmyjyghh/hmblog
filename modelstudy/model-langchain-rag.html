    # RAG

    1. RAG(Retrieval Augmented Generation)-检索增强⽣成

    将⼤模型应⽤于实际业务场景时会发现，通⽤的基础⼤模型基本⽆法满⾜我们的实际业务需求，主要有以下⼏⽅⾯原因

    • LLM的知识不是实时的，不具备知识更新
    • LLM可能不知道你私有的领域/业务知识
    • LLM有时会在回答中⽣成看似合理但实际上是错误的信息
    1. 为什么会⽤到RAG?

    1、提⾼准确性: 通过检索相关的信息，RAG可以提⾼⽣成⽂本的准确性。 2、减少训练成本：与需要⼤量数据来训练的⼤型⽣成模型相⽐，RAG可以通过检 索机制来减少所需的训练数据量，从⽽降低训练成本。 3、适应性强：RAG模型可以适应新的或不断变化的数据。由于它们能够检索最新 的信息，因此在新数据和事件出现时，它们能够快速适应并⽣成相关的⽂本。

    1. RAG VS Fine-tuning
    • RAG（检索增强⽣成）是把内部的⽂档数据先进⾏embedding，借助检索先获得⼤致的知识范围答案，再结合prompt给到LLM，让LLM⽣成最终的答案
    • Fine-tuning（微调）是⽤⼀定量的数据集对LLM进⾏局部参数的调整，以期望
    • LLM更加理解我们的业务逻辑，有更好的zero-shot能⼒。

    RAG 与 Fine-tuning 对比

    • RAG的优势在于它不需要⼤量的训练数据，只需要⼀定量的⽂档数据即可。这使得RAG模型更加轻量级，成本更低。
    • Fine-tuning的优势在于它可以使模型在特定任务上表现更好，因为它是在预训练模型的基础上进行的。
    • RAG的劣势在于它依赖于外部的检索机制，检索机制的质量直接影响到最终的答案质量。
    • Fine-tuning的劣势在于它需要⼤量的训练数据，训练数据的质量直接影响到模型的性能。

    # LangChain 框架

    # 核心组件

    LangChain之Model I/O 可以把对模型的使用过程拆解成三块: 输入提示(Format)、调用模型(Predict)、输出解析(Parse)

    1.提示模板: LangChain的模板允许动态选择输入，根据实际需求调整输入内容，适用于各种特定任务和应用。 2.语言模型: LangChain 提供通用接口调用不同类型的语言模型，提升了灵活性和使用便利性。 3.输出解析: 利用 LangChain 的输出解析功能，精准提取模型输出中所需信息，避免处理冗余数据，同时将非结构化文本转换为可处理的结构化数据，提高信息处理效率。

    1. Chain， 链中，输出解析器： StrOutputParser() JsonOutputParser()
    2. PromptTemplates --->>> ChatPromptTemplate

    Chain 链

    LangSmith LangSmith 是 LangChain 生态中一个极其重要的工具，它提供了一个平台，用于开发、测试、监控和部署基于 LangChain 的应用程序。

    下面我将为您全面介绍 LangSmith：它是什么、为什么需要它、如何安装使用以及最佳实践。

    # 一、LangSmith 是什么？

    LangSmith 是一个统一的开发、调试、测试、监控和评估 LLM 应用的平台。 你可以把它想象成 LangChain 应用的 “操作系统” 或 “控制中心”。

    它不是 LangChain 框架里的一个 Python 库，而是一个由 LangChain Inc. 提供的云端 SaaS 平台（也提供本地部署方案）。它的核心功能是追踪（Tracing） 你的 LLM 应用执行过程，并提供可视化界面。

    # 二、为什么需要 LangSmith？解决了什么痛点？

    在没有 LangSmith 之前，开发 LLM 应用像是在一个“黑盒”中调试：

    1. 调试困难： 链（Chain）或代理（Agent）的调用流程复杂，出错时很难定位是哪个环节（LLM 调用、工具执行、提示词问题）出了问题。
    2. 提示词迭代低效： 修改提示词后，需要重新运行整个流程才能看到效果，没有 A/B 测试和版本对比。
    3. 评估缺失： 如何知道新版本的 Chain 比旧版本更好？缺乏客观的、自动化的评估手段。
    4. 监控空白： 生产环境中的应用表现如何？哪些调用失败了？延迟和成本是多少？缺乏监控面板。

    LangSmith 完美地解决了以上所有问题。

    # 三、核心功能与使用场景

    功能 描述 使用场景
    ** tracing (追踪)** 自动记录每次 Chain、Agent、LLM、Tool 的调用详情，包括输入、输出、延迟、token 用量、成本等。 调试开发： 逐步查看应用执行流程，精准定位错误和性能瓶颈。
    调试与比较 将不同版本的提示词、模型或链的输入输出并排对比。 提示词工程： 快速迭代和测试不同提示词的效果，选择最佳版本。
    测试与评估 使用数据集（Dataset）批量运行你的链，并利用 LLM 作为评判员（Judge）进行自动评估。 版本控制： 在部署新模型或链之前，确保其性能优于旧版本。
    监控与报警 监控生产环境应用的性能、错误率、延迟等指标，并设置报警。 生产运维： 实时了解应用健康状态，及时发现和处理问题。
    协作 团队共享追踪记录、数据集和项目，共同迭代应用。 团队开发： 方便团队成员审查流程、共享知识。

    # 四、安装与快速上手

    # 3. 基本使用：自动追踪

    最简单的方式：什么都不用做！ 一旦设置了环境变量，LangChain 的大部分组件（LLMs, Chains, Agents, Tools）都会自动将执行记录发送到 LangSmith。

    import os
from langchain.llms import OpenAI
from langchain.agents import load_tools, initialize_agent, AgentType

# 设置环境变量（如果在脚本内设置）
os.environ["LANGCHAIN_TRACING_V2"] = "true"
os.environ["LANGCHAIN_API_KEY"] = "lsv2_pt_..."
os.environ["OPENAI_API_KEY"] = "sk-..." # 你的 OpenAI KEY

# 正常构建你的应用
llm = OpenAI(temperature=0)
tools = load_tools(["serpapi", "llm-math"], llm=llm)
agent = initialize_agent(tools, llm, agent=AgentType.ZERO_SHOT_REACT_DESCRIPTION, verbose=True)

# 运行 agent，执行过程会自动被 LangSmith 记录
agent.run("奥尼尔在哪支球队赢得了他的第一个NBA总冠军？")

    现在，运行这个脚本，然后刷新你的 LangSmith 主页，你就能看到这次运行的完整记录。

    # 五、部署与生产环境考虑

    1. 网络与延迟： LangSmith 是云端服务，你的应用需要能访问外网。追踪调用是异步非阻塞的，不会影响主应用的性能，但需要考虑网络抖动。
    2. 数据隐私： 如果你的数据极其敏感，需要询问 LangSmith 的合规性，或者等待/寻求本地部署（On-Premise） 方案。他们已面向企业提供此服务。
    3. 成本： LangSmith 有免费额度，超出后按调用次数收费。需要评估成本效益。
    4. 监控报警设置： 在生产环境中，务必在 LangSmith 项目中设置监控和报警规则（如错误率 > 1% 时发送邮件/Slack 通知）。

    # 总结：作为工程师，你应如何看待 LangSmith？

    • 开发阶段： 它是你的 “超级调试器”。抛弃 verbose=True 那种简陋的输出，拥抱可视化的、可回溯的调试体验。
    • 测试阶段： 它是你的 “评估框架”。用它来科学地、数据驱动地验证你的应用变更是否有效。
    • 生产阶段： 它是你的 “监控仪表盘”。实时洞察应用性能、成本和异常。

    强烈建议：从现在开始，在你所有的 LangChain 项目中，从一开始就集成 LangSmith。它会极大地提升你的开发效率和应用的可靠性。

    # LangSmith 核心功能全景图

    # 1. Tracing & Debugging（追踪与调试） - 最核心功能

    作用：自动记录LLM应用的每一次调用细节，形成可视化的执行链。

    功能细节：

    • 全链路追踪：记录每个组件的输入/输出（LLM调用、工具执行、提示词渲染）
    • 时间线视图：显示每个步骤的耗时，快速定位性能瓶颈
    • 详细元数据：查看token用量、成本、模型参数等
    • 比较运行：并排对比不同配置的执行结果

    # 2. Testing & Evaluation（测试与评估）

    作用：系统化地测试和评估您的应用表现。

    功能细节：

    • 数据集管理：创建和管理测试用例集合
    • 批量测试：在多个输入上运行您的链/代理
    • 自动评估：使用LLM作为评判员或自定义评估函数
    • 指标分析：准确率、延迟、成本等关键指标可视化

    # 3. Monitoring & Alerting（监控与告警）

    作用：监控生产环境应用的性能和健康状况。

    功能细节：

    • 实时仪表盘：监控错误率、延迟、吞吐量等
    • 自定义告警：设置阈值，自动发送通知（邮件、Slack等）
    • 生产分析：识别性能退化和异常模式
    • 使用量统计：跟踪API调用次数和成本

    # 4. Prompt Management（提示词管理）

    作用：优化和管理提示词版本。

    功能细节：

    • 版本对比：A/B测试不同提示词的效果
    • 提示词注册：将优秀提示词保存为可复用模板
    • 协作分享：团队共享和评审提示词

    # 5. Collaboration & Organization（协作与组织）

    作用：支持团队开发和知识共享。

    功能细节：

    • 项目管理：按项目组织追踪记录和数据集
    • 团队管理：成员权限和访问控制
    • 知识共享：共享成功的提示词和配置

    # 实用技巧和最佳实践

    # 1. 高效调试技巧

    # 添加自定义 metadata 便于筛选
from langchain.callbacks.manager import tracing_v2_enabled

with tracing_v2_enabled(
    project_name="my-project",
    tags=["experiment", "v2"],
    metadata={"version": "1.0", "user_id": "123"}
):
    result = chain.invoke("your question")

    # 2. 快速筛选和搜索

    • 使用 tags: 筛选特定标签的运行
    • 使用 metadata.key:value 搜索自定义元数据
    • 使用 duration:>5000 查找耗时长的运行

    # 3. 评估策略

    # 自定义评估函数
def contains_keyword_evaluator(run, example):
    """检查输出是否包含关键信息"""
    output = run.outputs["output"]
    expected_keywords = example.outputs.get("expected_keywords", [])
    
    score = any(keyword.lower() in output.lower() 
               for keyword in expected_keywords)
    
    return {"score": score, "comment": f"Contains keywords: {expected_keywords}"}

    # 常见工作流示例

    # 工作流1：提示词迭代优化

    1. 用初始提示词运行测试集
    2. 分析失败案例，修改提示词
    3. 用新提示词重新运行，比较结果
    4. 重复直到满意效果

    # 工作流2：生产问题排查

    1. 收到用户反馈回答质量下降
    2. 在LangSmith中筛选最近24小时的运行
    3. 按错误率或低分筛选，找到问题模式
    4. 分析具体trace，定位问题根源

    # 工作流3：性能优化

    1. 识别耗时最长的组件
    2. 优化提示词或调整模型参数
    3. 添加缓存机制
    4. 监控优化后的性能指标
