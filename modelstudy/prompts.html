    # 2. 思维链提示(COT)

    • 它用于促进 LLM在解决问题或推理任务中展示其思考过程。
    • 它鼓励模型在生成回答时, 不仅提供最终答案, 还输出一系列中间步骤或思维过程, 以增加透明度和解释性。

    示例：

    # 关联问题：如果Chain of Thought(COT)提示失败, 如何提高LLM的推理能力?

    COT 提示是一种贪心解码策略, 它通过选择每一步中最有可能的下一个词来生成逻辑序列。

    • 然而,这种方法只考虑局部最优解, 而不考虑整体序列的连贯性。

    • Self-consistency(自我一致性) 是一种新的解码策略,可以替代COT提示中的贪心解码, 以提高推理能力。

    总结一下, 如果Chain of Thought提示在推理任务中失败,

    • 可以通过使用Self-consistency策略来改进。

    • 该策略通过采样多个推理路径和选择最一致的答案, 能够克服贪心解码的局限性, 进而提高模型的推理表现。

    • 虽然这种方法会增加一定的计算成本, 但它能够有效提高推理任务的准确性。

    # 3. ReAct Prompting, 以及它的优势？

    ReAct Prompting 是Agent架构中非常重要的一种方法

    • 它的核心目的是：打通 LLM的思考能力 和 行动能力 之间的断层

    为什么需要ReAct Prompting？

    eg: 需要地图来规划路线、方向盘来控制方向，实际驾驶

    • 模型既要知道怎么想, 也要知道怎么做, 还能根据路况实时调整路线
    • 这就是： 思考、行动、观察

    ReAct Prompting 的重要性，主要体现在以下几个点

    1. 推理与行动的协同(Synergy)

    ReAct 的最大特点就是让模型：在每一步推理的基础上决定下一步行动， 并根据行动得到的结果：继续更新接下来的推理路径。

    • 同步思考 + Act
    • 能让模型像人一样规划流程、执行任务 & 应对中间可能出现的异常情况。
    1. 与外部环境高效互动(Enhanced Interactivity)
    • ReAct Prompting，允许LLM实时对接这些外部资源, 比如搜索引擎、计算器、数据库等, 动态获取新信息。
    1. 更强的人类可解释性与可信度 (Interpretability & Trustworthiness)

    ReAct 的推理链和行动步骤是结构化且可追踪的。

    • 它可以逐步输出"我为什么要做这个动作"、"这个动作得到了什么结果"、"我接下来应该怎么么改", 这种方式更容易被人类理解和信任。
    1. 在决策类任务中表现优异(Superior Decision-Making)
    • 尤其在需要复杂判断、多轮计划、调用工具的任务中,
    • ReAct的表现往住优于传统Chain-of-Thought或Tool-Calling方案。
    • 在很多交互式决策类 benchmark中, 它的效果显著领先。

    ReAct Prompting 框架是如何运作的？

    模拟人类在处理问题时的一种自然行为, 也就是： 思考(Thought) --> 行动(Act) --> 观察(Observation)

    这个流程可以不断循环，直到任务完成。

    # 4. ICL(In-Context-Learning)

    允许大模型训练完成后，通过提示和上下文学习适应特定任务和情境

    • 通过任务提示+示例，引导模型完成任务

    工作原理

    • 提示词里，包含自然语言，指令，示例 和 上下文信息，帮助模型理解任务的目标
    • 从示例中学习，通过分析示例，学习和执行类似的任务

    # 5. 有哪些策略，可以写出好的提示词?

    • 尽可能详细地描述，任务，指令，角色
    • 通过示例或者明确的指令
    • 提供高质量的示例数据
    • 重复关键词或短语，如春天
    • 将一个大提示分解成更小，更简单的提示

    # 6. COT (Chain-of-Thought Prompt) 和 Instruction Tuning

    CoT提示通过在prompt中引间推理步骤, 启发了模型的复杂推理能力。

    • CoT要求模型在生成最终答案之前, 首先生成一个或多个中间计算结果。即再给出答案之前生成一系列简短的句子来逐步描述推理逻辑, 这些逻辑被称为推理链或理由链,最终引导模型给出最终答案。

    • 针对于大型模型(参数大于50B)和复杂的推理任务, COT的性能提升较为明显。而对于简单任务,CoT提示的性能提升较小。

    • 在原论文中,复杂的推理任务指:算术推理、常常识推理、符号推理。

    Instruction Tuning

    Instruction Tuning(指令微调), 是一种更加高级的优化技术。

    • Instruction Tuning是一种通过，在 由(指令, 输出)对组成的数据集 上进一步训练LLMs的过程。
    {
    "instruction": "你是xxx",
    "answer": "xxx"
}

    • 其中,指令代表模型的人类指令,输出代表遵循指令的期望输出。

    • 这个过程有助于弥合LLMs的下一个词预测目标与用户让LLMs遵循人类指令的目标之间的差距。

    • 指令微调可以被视为有监督微调的一种特殊形式, 专注于通过理解和遵循人类指令来增强大型语言模型的能力和可控性。

    • SFT是一种使用标记数据对预训练模型进行微调的过程,以便模型能够更好地执行特定任务。

    • 而指令微调是一种: 通过在包括(指令,输出)对的数据集上进一步训练大型语言模型(LLMs)的过程, 以增强LLMs的能力和可控性。

    • 指令微调的特殊之处在于其数据集的结构, 即由人类指令和期望的输出组成的配对。

    • 这种结构使得指令微调专注于让模型理解和遵循人类指令。

    # 关联问题：请阐述一下instruction tuning和prompt learning的区别？

    # 7. Prompt design, Prompt tuning, fine tuning的区别

    微调(Finetuning) 是一种常用的技术, 许多技术(如LORA) 可以使微调更加快速和高效。

    • 参数高效微调 然而, 微调依然是一项复杂的工作, 例如,需要理解相关技术并准备数据。通常情况下,SFT可以运作得很好, 但是, 效果更好的RLHF模型的训练需要高度的技巧。

    • prompt design 只需要对prompt进行调整, 这也是一种非常有效的方法。

      • zero-shot
      • few-shot

    one-shot和few-shot 通过在prompt中给出问题解答的例子, 可以让模型参考这些例子来解答问题, 这样也可以非常好地使LLM处理各种新任务。

    • Prompt tuning

    它的核心思想是：冻结主模型参数，在训练数据前，加入一小段Prompt（只通过优化一小段可学习的“软提示”（Soft Prompt）），来激发模型解决特定任务的潜能。

    • 只训练Prompt 的表示层，也就是一个Embedding 模块。

    • 其中， Prompt 有两种形式， 一种是: hard prompt ， 一种是: soft prompt

    # 动态prompt

    动态prompt是一种根据条件、变量或外部输入自动调整的提示词技术，能让AI生成更个性化、更精准的内容。

    # 1. 什么是动态prompt？

    • 不是固定的文本，而是包含变量、条件判断、模板的提示结构
    • 根据用户输入、上下文或参数实时生成最终提示词
    • 提高提示的复用性和适应性

    # 2. 主要应用场景

    • 批量生成：为不同对象生成类似但定制化的内容
    • 个性化交互：根据用户画像调整回答风格
    • 多模态切换：根据输入决定输出格式（文本、代码、表格等）
    • A/B测试：快速试验不同提示词变体

    # 实现动态prompt的常见方法

    # 模板变量替换

    # 简单示例
template = "为{product}写一段{style}风格的广告文案，突出{feature}亮点"
variables = {
    "product": "智能手表",
    "style": "科技感",
    "feature": "健康监测"
}
# → 最终prompt: "为智能手表写一段科技感风格的广告文案，突出健康监测亮点"

    # 1. 角色模式

    # 角色设定

# 任务

    # 2. 结构化输出模式

    请以JSON格式输出，包含以下字段：
- summary: 简短总结
- key_points: 数组，列出3-5个要点
- action_items: 建议的行动步骤

    # 3. 逐步推理模式

    让我们一步步思考：
1. 首先，分析问题的核心是什么
2. 然后，考虑相关因素
3. 接着，评估可能的方法
4. 最后，给出结论

    # 提升模型 Faithfulness 的 Prompt 调优示例

    覆盖基础约束、细节强调、错误规避三种核心场景：

    # 1. 基础通用型（适用于日常信息处理）

    “请基于我提供的内容[此处插入具体内容]进行[总结/分析/改写]，核心要求如下：

    1. 严格遵循原文事实，不添加任何未提及的信息；
    2. 若原文存在模糊表述，直接保留或标注‘信息未明确’，不主观推测补充；
    3. 输出后需自查是否有偏离原文的内容，确保完全忠实于输入信息。”

    # 2. 细节强化型（适用于专业资料/数据类任务）

    “针对以下数据[此处插入数据表格/专业文本]，请完成[数据解读/要点提取]任务。务必做到：

    • 所有结论均需对应原文具体依据，标注依据所在的段落/数据位置；
    • 涉及数字、术语、逻辑关系的内容，与原文完全一致，不做任何简化或调整；
    • 若发现原文存在矛盾，仅指出矛盾点，不自行修改或统一矛盾内容。”

    # 3. 错误规避型（适用于易编造信息的场景）

    “请回答关于[具体主题，如某历史事件/产品参数]的问题，回答时必须满足：

    1. 仅使用我提供的参考资料[插入参考资料]中的信息，不调用任何外部知识；
    2. 若参考资料无法覆盖问题答案，直接回复‘参考资料未包含该信息’，不编造相关内容；
    3. 对参考资料中的不确定表述（如‘可能’‘大概’），需完整保留，不转化为肯定或否定表述。”
