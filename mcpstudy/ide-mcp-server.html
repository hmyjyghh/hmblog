    IDE 使用MCP Server实操

    寒梦

    # IDE 使用MCP Server实操

    # 1. VSCode

    # 1.1 安装Cline 插件

    1. 安装Cline 插件
    2. 打开Cline 插件, 点击设置
    3. 搜索MCP Server, 点击添加MCP Server
    4. 填写MCP Server 名称, 填写MCP Server 地址, 比如http://localhost:8080
    5. 点击添加

    # 1.2 好用的MCP Server 插件

    • time
    • browser-tools-mcp
    • GitHub

    # 2. Cursor

    • 同理

    # 3. Trae

    1. Trae 本身就继承了@Builder 和 @Builder with MCP
    2. AI功能管理-MCP, 自行添加MCP Server
    3. AI功能管理-智能体

    # 3.1 AI功能管理-智能体

    自定义智能体

    1. MCP 里面添加 Git 和 GitHub 这2个 MCP Server
    2. AI功能管理-智能体, 自定义智能体
    3. 智能体-自定义智能体, 填写智能体名称, 填写提示词, 比如你是一个Git 专家，擅长使用MCP Server 操作Git 仓库与使用Github,工具里勾选需要的MCP Server, 比如Git 和 GitHub
    4. 在AI对话框里，使用@符号，选择这个智能体, 输入相关的命令, 比如帮我将当前文件夹下的所有文件添加到Git 仓库

    # 3.2 使用rules

    1. AI 功能管理-规则,

    以项目规则为例,创建project_rules.md 文件 比如让AI实现， 你画我猜网页游戏 project_rules.md文件内容如下：

    ##项目概述
##技术栈选择
- 脚手架:Vite
- 前端:React +TypeScript
- 画布:HTML5 Canvas
- 后端:Node.js +Express
- AI 集成:0penAI Vision API
    1. 使用 @builder #键盘选 rules文件：project_rules.md
    2. 对话内容, 填入：请根据rules 里面的介绍，拆解任务，任务可以跟踪状态，把所有任务写到task.md 当中去
    3. 根据task.md , 用#去选，进行逐项开发，并且标记状态
    4. 根据AI指令去接受或者执行
    5. 完成各项Task

    # 3.2 @Builder 对实际企业项目带来的赋能/提效

    1. 打开所在项目
    2. 创建project_rules.md 文件后，填入以下内容
    # 项目规则

# step1 你先读一遍 src\views\dev\dev-list.vue 这个文件

# step2 然后你再读一遍 src\views\dev\dev-list-v2.vue 这个文件

# step3 你需要对比这两个文件的差异，分析出其中的不同点

# step4 我需要你根据不同点，参照这个改动, 修改 @/views/p-order-list.vue 这个文件

## 至于修改p-order-list.vue 这个文件，我需要你按照步骤做以下事情
1. 你需要在p-order-list.vue 这个文件的目录下，直接建一个p-order-list-v2.vue的文件
2. 你需要根据 step3 中分析出的不同点，修改 p-order-list.vue 这个文件, 然后将修改后的内容复制到p-order-list-v2.vue这个文件中


## 限制：
- 切记不要改p-order-list.vue 这个文件的任何内容
    1. 打开dev-list.vue 文件, 输入@Builder with MCP 指令, 会弹出一个对话框, 选择project_rules.md 文件, 等待 Trae 完成修改
